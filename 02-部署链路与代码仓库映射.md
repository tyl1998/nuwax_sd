## 1. 直接部署链路

从以下文件可以确认当前部署主链：

- `../nuwax_deploy/docker/docker-compose.yml`

### 1.1 主服务部署链

`nuwax_deploy/docker/docker-compose.yml` 里的核心服务：

- `frontend`
- `backend`
- `mcp-proxy`
- `mysql`
- `redis`
- `milvus`

对应代码仓库映射：

| 部署服务 | 代码仓库 | 说明 |
| --- | --- | --- |

### 1.2 Agent Computer / Sandbox 部署链

`nuwax_computer_deploy/docker-computer/docker-compose.yml` 里的核心服务：

- `rcoder`
- `agent-runner-image-pre-pull`
- `agent-runner-image-puller`

从 `rcoder/README.md` 可知，`rcoder` 是主控层，会动态创建 `agent_runner` 容器，并对外暴露：

- `/computer/chat`
- `/computer/progress/{session_id}`
- `/computer/vnc/...`
- `/computer/audio/...`
- `/computer/ime/...`

对应代码仓库映射：

| 部署服务 | 代码仓库 | 说明 |
| --- | --- | --- |
| `rcoder` | `rcoder` | 沙箱主控、容器调度、gRPC/SSE 桥接、代理路由 |
| VNC 前端能力 | `noVNC` | 远程桌面网页端 |

## 3. 整体链路理解

可以把当前工程理解成 4 层：

```mermaid
flowchart LR
    A[Web/Mobile/Desktop Client] --> B[nuwax / nuwax-mobile / nuwaclaw]
    B --> C[nuwax-backend]
    C --> D[mcp-proxy]
    C --> E[(MySQL)]
    C --> F[(Redis)]
    C --> G[(Milvus)]
    C --> H[rcoder]
    H --> I[agent_runner containers]
    H --> J[noVNC]
    I --> K[nuwaxcode / claude-code-acp-ts]
    I --> L[run_code_rmcp]
    B --> M[nuwax-file-server]
```

### 分层解释

- 交互层：`nuwax`、`nuwax-mobile`、`nuwaclaw`
- 平台层：`nuwax-backend`
- 协议与桥接层：`mcp-proxy`
- 沙箱与执行层：`rcoder`、`agent_runner`、`noVNC`、`nuwaxcode`、`claude-code-acp-ts`、`run_code_rmcp`

## 4. 本轮建议纳入“必看仓库”的范围

按照“部署直接出现 + 运行时明确依赖 + 便于串整体链路”的原则，建议先聚焦这 11 个仓库：

- `nuwax`
- `nuwax-backend`
- `mcp-proxy`
- `rcoder`
- `noVNC`
- `nuwaclaw`
- `nuwax-file-server`
- `nuwaxcode`
- `claude-code-acp-ts`
- `run_code_rmcp`
- `nuwax-cli`

其中：

- `nuwax-cli` 负责部署、升级、备份，是“运维入口”
- `nuwax-mobile` 是端侧补充，建议第二阶段再看
- 模板/设计类仓库放到最后
