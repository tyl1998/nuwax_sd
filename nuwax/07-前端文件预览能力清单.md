# 前端文件预览能力清单

## 1. 概述

前端的文件预览能力主要分布在两个场景：

- **文件树右侧面板**（`FileTreeView`）：用户点击文件树中的文件时，在右侧区域预览
- **聊天消息区域**（`ChatView` + `MarkdownRenderer`）：AI 回复中的 Markdown、代码、图表等渲染

核心组件：

- [`FilePreview`](../../../nuwax/src/components/business-component/FilePreview/index.tsx)：通用文件预览组件
- [`FileTreeView`](../../../nuwax/src/components/FileTreeView/index.tsx)：文件树 + 内容预览容器
- [`CodeViewer`](../../../nuwax/src/components/CodeViewer/index.tsx)：代码编辑/查看器（Monaco Editor）
- [`MarkdownRenderer`](../../../nuwax/src/components/MarkdownRenderer/index.tsx)：Markdown 流式渲染
- [`PagePreviewIframe`](../../../nuwax/src/components/business-component/PagePreviewIframe/index.tsx)：页面预览 iframe
- [`VncPreview`](../../../nuwax/src/components/business-component/VncPreview/index.tsx)：远程桌面预览

---

## 2. 文件树预览（FileTreeView → renderContent）

文件树中点击文件后，根据文件类型走不同的渲染分支：

### 2.1 已支持的预览类型

| 文件类型 | 匹配方式 | 渲染组件 | 说明 |
|---------|---------|---------|------|
| **图片** (jpg/png/gif/webp/svg/bmp/ico) | `isImageFile()` | `FilePreview` (fileType="image") | 支持单图和画廊模式 |
| **视频** (mp4/webm/mov/avi/mkv) | `isVideoFile()` | `FilePreview` (fileType="video") | HTML5 video 播放器 |
| **音频** (mp3/wav/ogg/m4a/aac/flac) | `isAudioFile()` | `FilePreview` (fileType="audio") | HTML5 audio 播放器 |
| **Office 文档** (docx/doc/xlsx/xls/pptx/ppt/pdf) | `isDocumentFile()` | `FilePreview` | 使用第三方库渲染 |
| **文本/代码** (txt/json/xml/js/ts/css/py/java/go/sql/yaml/csv 等 30+ 种) | 默认分支 | `CodeViewer`（Monaco Editor） | 语法高亮 + 只读/编辑 |
| **HTML** (html/htm) | 正则 `/\.html?($\|\?)/i` | `FilePreview` (fileType="html") | sandbox iframe 渲染 |
| **Markdown** (md/markdown) | `isMarkdownFile()` | `FilePreview` (fileType="markdown") | 使用 `PureMarkdownRenderer` 渲染 |
| **VNC 远程桌面** | SSE 事件触发 | `VncPreview` | noVNC iframe |

### 2.2 FilePreview 内部渲染分支

`FilePreview` 组件内部根据 `FileType` 走不同渲染路径：

| FileType | 渲染方式 | 依赖库 |
|----------|---------|-------|
| `docx` | `@js-preview/docx` | jsPreviewDocx |
| `xlsx` | `@js-preview/excel` | jsPreviewExcel |
| `pdf` | `@js-preview/pdf` | jsPreviewPdf |
| `pptx` | `pptx-preview` | pptxPreview |
| `image` | `<img>` 标签 | 原生 |
| `audio` | `<audio>` 标签 | 原生 |
| `video` | `<video>` 标签 | 原生 |
| `html` | sandbox `<iframe>` | 原生 |
| `markdown` | `PureMarkdownRenderer` | ds-markdown |
| `text` | `<pre><code>` 纯文本 | 原生 |

### 2.3 不支持预览的文件

当文件类型为 `unsupported` 时，显示"不支持预览"占位图标，可选显示下载按钮。

## 3. 聊天消息渲染（MarkdownRenderer）

AI 回复通过 `MarkdownRenderer` 渲染，基于 `ds-markdown` 库，支持：

| 能力 | 实现方式 |
|------|---------|
| **Markdown 基础语法** | ds-markdown 内置 |
| **代码块 + 语法高亮** | ds-markdown 内置 + highlight.js |
| **Mermaid 图表** | `mermaidPlugin` 插件 |
| **LaTeX 数学公式** | `katexPlugin` 插件（KaTeX） |
| **自定义 block**（工具调用结果、卡片等） | `genCustomPlugin` 插件 |
| **流式打字效果** | `MarkdownCMD` + rAF + 30ms interval |
| **思考过程折叠** | 可折叠的 thinking 块 |
| **图片渲染** | `OptimizedImage` 组件 |

---

## 4. 页面预览（PagePreviewIframe）

SSE `PROCESSING` 事件中 `type === 'Page'` 时触发，右侧弹出 iframe 预览：

- 支持 HTML 和 Markdown 两种 `data_type`
- 支持历史记录后退/前进
- 支持刷新、复制链接
- sandbox 隔离

---

## 5. 远程桌面预览（VncPreview）

SSE `PROCESSING` 中出现 `OPEN_DESKTOP` 事件时触发：

- 通过 noVNC iframe 实现
- 支持空闲检测 + 自动断开
- 支持只读模式
- 支持 URL 重试机制

## 6. 技能（Skill）文件预览的现状与不足

### 6.1 当前技能文件的加载方式

技能文件通过 [`fetchContentFromUrl()`](../../../nuwax/src/services/skill.ts) 获取内容，该函数：

- 从 `/api/skill/...` 或 `/api/computer/static/...` 等接口获取文件文本内容
- 返回纯文本字符串
- 不走浏览器 HTTP 缓存（`cache: 'no-store'`）

### 6.2 技能文件预览的问题

在 `FileTreeView` 中，技能文件（`isProjectSkill=true`）的预览逻辑：

1. **Markdown 文件**（如 `SKILL.md`）：通过 `FilePreview` 组件的 `content` 属性传入文本，使用 `PureMarkdownRenderer` 渲染
2. **代码文件**：通过 `CodeViewer`（Monaco Editor）渲染
3. **其他文件**：如果 `isPreviewableFile()` 返回 false，则无法预览

### 6.3 已知不足

| 问题 | 说明 |
|------|------|
| **Markdown 渲染样式可能混乱** | `PureMarkdownRenderer` 使用的是聊天场景的样式，在文件预览场景下可能不适配 |
| **非文本文件无法预览** | 技能中如果包含图片、PDF 等二进制文件，当前 `fetchContentFromUrl` 只返回文本，无法处理 |
| **大文件性能** | Monaco Editor 加载大文件可能卡顿 |
| **缺少文件类型图标映射** | 部分文件类型在文件树中没有对应的图标 |

## 7. 文件类型判断工具函数

位置：[`src/utils/appDevUtils.ts`](../../../nuwax/src/utils/appDevUtils.ts)

| 函数 | 作用 |
|------|------|
| `isImageFile(fileName)` | 判断是否为图片文件 |
| `isVideoFile(fileName)` | 判断是否为视频文件 |
| `isAudioFile(fileName)` | 判断是否为音频文件 |
| `isDocumentFile(fileName)` | 判断是否为 Office 文档 |
| `isPreviewableFile(fileName, includeCode?)` | 判断是否可预览 |
| `isMarkdownFile(fileName)` | 判断是否为 Markdown 文件 |

| `CONTENT_TYPE_MAP` | Content-Type → FileType 映射 |

## 8. 总结

前端目前的文件预览能力覆盖面较广，主要包括：

1. **文档类**：DOCX、XLSX、PPTX、PDF（通过第三方 js-preview 库）
2. **媒体类**：图片、音频、视频（原生 HTML5 标签）
3. **代码类**：30+ 种编程语言（Monaco Editor 语法高亮）
4. **Markdown**：通过 ds-markdown 渲染，支持 Mermaid、KaTeX
5. **HTML**：sandbox iframe 渲染
6. **页面预览**：iframe 嵌入，支持历史导航
7. **远程桌面**：noVNC iframe

主要短板在于：

1. 技能文件场景下 Markdown 渲染样式可能不适配文件预览场景
2. 二进制文件（图片、PDF 等）在技能文件树中无法通过文本接口获取
3. 缺少针对技能文件的专用预览优化
