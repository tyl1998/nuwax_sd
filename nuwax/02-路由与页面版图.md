# 路由与页面版图

## 1. 路由总入口

先看 [src/routes/index.ts](../../nuwax/src/routes/index.ts)。

最上层可以先分成两类页面：

### 独立页

- `/login`
- `/verify-code`
- `/set-password`
- `/chat-temp/:chatKey`

特点：

- `layout: false`
- 不走主布局

### 主应用页

- `/`

特点：

- 走 [src/layouts/index.tsx](../../nuwax/src/layouts/index.tsx)
- 外层包了 [src/wrappers/authWithLoading.tsx](../../nuwax/src/wrappers/authWithLoading.tsx)

这说明大部分业务页面都属于“登录后主应用”。

## 2. 首页入口不是静态首页

[src/pages/Index/index.tsx](../../nuwax/src/pages/Index/index.tsx) 的逻辑很关键：

- 它会从菜单树里找第一个可用菜单
- 然后重定向过去

所以 `/` 不是传统 landing page，而是“基于菜单权限的动态首页入口”。

## 3. 页面版图怎么理解

从 `routes/index.ts` 和 `src/pages/` 目录一起看，这个仓库大致可以拆成 7 个前端子系统。

### A. 登录与会话入口

主要目录：

- `Login`
- `VerifyCode`
- `SetPassword`
- `ChatTemp`
- `Chat`
- `HistoryConversation`

作用：

- 登录
- 验证码
- 设置密码
- 临时对话
- 正式对话
- 历史会话

### B. 首页与智能体入口

主要目录：

- `Index`
- `Home`
- `AgentDetails`
- `MyComputerManage`

作用：

- 主页推荐与分类
- 默认智能体入口
- 智能体详情
- 电脑/沙箱相关入口

### C. 工作空间开发体系

主要目录：

- `Space`
- `SpaceDevelop`
- `EditAgent`
- `SpacePluginTool`
- `SpacePluginCloudTool`
- `SpaceResource/*`
- `SpaceKnowledge`
- `SpaceLibrary`
- `SpaceTable`
- `SpaceSkillManage`
- `SpaceMcpManage`
- `SpaceMcpCreate`
- `SpaceMcpEdit`
- `SpacePageDevelop`
- `SpaceTaskCenter`

这是整个工程最核心的一块。

它基本覆盖了：

- 智能体开发
- 插件管理
- 工作流管理
- MCP 管理
- 知识库
- 数据表
- 页面开发
- 技能管理
- 任务中心

### D. 广场与发布体系

主要目录：

- `Square`
- `SpaceSquare`
- `PublishedManage`
- `PublishAudit`
- `SkillDetails`

作用：

- 广场浏览
- 发布详情
- 审核
- 已发布内容管理

### E. 开放应用与商业化体系

主要目录：

- `OpenApp`
- `MorePage/*`

其中 `MorePage` 下可以明显看到：

- `ApiKey`
- `ApiKeyLogs`
- `MySubscriptions`
- `MyOrders`
- `MyEarnings`
- `CreditRecords`
- `UsageStats`
- `ModelPermissions`

说明这个前端已经覆盖：

- API 能力开放
- 订阅计费
- 订单
- 收益
- 用量统计

### F. 系统管理后台

主要目录：

- `SystemManagement/*`
- `UserManage`
- `TeamSetting`

这里包含：

- Dashboard
- 日志查询
- 菜单权限
- 模型监控
- 系统配置
- 任务管理
- 订阅积分

这部分明显是后台管理域。

### G. 高复杂度编辑器/AI 工作台

主要目录：

- `Antv-X6`
- `AppDev`
- `AppDevDesign`

这几个目录信号很强：

- `Antv-X6` 说明有工作流/图编排编辑器
- `AppDev` 说明有 Web IDE / AI Agent 开发辅助工作台
- `AGENTS.md` 也专门描述了 AI Agent / SSE / AppDev 架构

## 4. 路由组织的几个重要结论

### 工作空间是核心主轴

很多关键页面都挂在：

- `/space/:spaceId/...`

这意味着大部分核心业务都是“围绕空间展开”的。

### 广场和空间是两套视角

从路由能看出：

- `space` 视角：偏工作区、研发、资源管理
- `square` 视角：偏发布、展示、消费

### 系统管理是另一套后台域

- `/system/...`

这说明这个前端其实把“C 端/工作台/后台管理”都装进了一个工程。

## 6. 路由层面的总体判断

一句话总结：

`nuwax` 前端不是“单一产品页面”，而是把智能体平台、资源开发台、发布广场、商业化中心、系统后台和 AI 开发工作台整合在一起的超级前端工程。
