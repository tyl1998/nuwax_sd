# 技术栈与运行时

## 1. 核心技术栈

从 [package.json](../../nuwax/package.json) 可以直接看出，这个工程的核心栈是：

- `React 18`
- `TypeScript`
- `@umijs/max 4`
- `antd 5`
- `@ant-design/pro-components`
- `ahooks`
- `vitest`

增强能力还包括：

- `Monaco Editor`
- `@antv/x6` / `@antv/g6`
- `@tiptap/*`
- `zustand`
- `mermaid`
- `react-markdown`

这说明它不是纯管理后台，而是混合了：

- 管理台
- 富交互编辑器
- 工作流编排
- AI 对话
- Web IDE

## 2. 工程是怎么跑起来的

关键脚本都在 [package.json](../../nuwax/package.json)：

- 开发：`pnpm dev`
- 生产构建：`pnpm build:prod`
- 测试：`pnpm test`

本质上都是在调用 `max dev` / `max build`，也就是 Umi Max 的工作流。

## 3. Umi 配置层负责什么

重点看 [config/config.ts](../../nuwax/config/config.ts)。

它负责：

- 注册路由
- 打开 `antd`、`locale`、`model`、`request`、`initialState`
- 指定 `npmClient: 'pnpm'`
- 配置顶部 `headScripts`
- 配置打包时的静态资源复制与压缩

这说明这套前端的“框架底座”是 Umi，而不是手写 React Router + Vite 的轻量项目。

## 4. 一个很关键的运行时特征：PC/H5 双端跳转

[config/config.ts](../../nuwax/config/config.ts) 里有一大段 `headScripts` 注入脚本，核心作用是：

- PC 打开移动端 URL 时跳回 PC 页面
- 移动端打开 PC 页面时跳去 `/m/`
- 智能体详情、临时会话、应用详情等页面做双向映射

这说明：

1. `nuwax` 不只是 PC 管理台
2. 它和 H5/移动端地址体系是联动的
3. 前端层自己承担了一部分端间跳转适配逻辑

## 5. 运行时入口做了什么

重点看 [src/app.tsx](../../nuwax/src/app.tsx)。

最关键的是 `getInitialState()`：

- 初始化国际化
- 非登录页时拉取用户信息
- 根据用户信息同步语言
- 拉取菜单数据

这意味着：

- 菜单不是写死在前端里的
- 页面权限和菜单展示强依赖后端返回
- 前端刷新时会先做“用户 + 菜单”恢复

另外它还负责：

- 全局 chunk 加载失败提示
- 全局主题初始化
- 统一语言与主题注入
- 控制台输出版本号

## 6. 主布局负责什么

重点看 [src/layouts/index.tsx](../../nuwax/src/layouts/index.tsx)。

主布局主要负责：

- 菜单栏
- 移动端与桌面端切换
- 一级/二级菜单宽度与展开状态
- 页面主容器
- 主题样式类

它还会在启动时加载：

- 菜单数据
- 广场分类
- 工作空间列表

所以它不是“纯展示壳子”，而是前端全局工作区的装配中心。

## 7. 鉴权 wrapper 负责什么

重点看 [src/wrappers/authWithLoading.tsx](../../nuwax/src/wrappers/authWithLoading.tsx)。

它负责：

- 页面进入前检查登录态
- 用 `sessionStorage` 缓存登录状态
- 处理登录页、验证码页、设置密码页、临时对话页的豁免
- 拉取租户配置
- 根据租户配置判断当前页面是否隐藏

这说明这里的权限控制至少分三层：

1. 是否登录
2. 是否有菜单权限
3. 是否被租户配置禁用

## 8. 从运行时角度如何理解这个项目

可以把它拆成四层：

### 框架层

- `Umi Max`
- `Ant Design`
- `request` / `model` / `initialState`

### 全局运行时层

- `app.tsx`
- `layouts`
- `auth wrapper`
- `theme / i18n`

### 页面系统层

- `src/pages/*`

### 接口与业务层

- `src/services/*`
- hooks
- model
- utils

一句话总结：

这个工程的运行时不是“React 页面组件自己跑”，而是先经过 Umi 配置、全局状态恢复、鉴权、布局装配，再进入具体业务页面。
