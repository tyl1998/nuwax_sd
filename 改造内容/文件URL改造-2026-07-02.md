# 文件 URL 改造 - 2026-07-02

## 问题背景

通过 IP 地址访问平台时（如 `http://10.94.77.42:18080`），出现两个问题：

1. **CORS 403**：后端 `CORS_ALLOW_ORIGIN` 只配置了 `localhost`，未包含 IP 地址
2. **后端内部文件访问超时**：后端生成的文件 URL 使用 `siteUrl`（如 `http://10.94.77.42:18080`），后端自己用该 URL 发 HTTP 请求解析文档时，访问不到外部地址导致超时

## 改造方案

### 核心思路

- 文件 URL **仍然存储完整 URL**（前端知识库模块依赖 `url` 字段为完整 URL，改为相对路径会导致上传流程中断）
- 后端内部访问文件时，通过 `FileUrlResolver` 将外部地址转为 `http://localhost:{port}`，自己访问自己
- `FileUrlResolver` 对已经是绝对 URL 的请求，将 host 替换为 localhost；对相对路径直接拼接 localhost

### 新增文件

| 文件 | 说明 |
|------|------|
| `FileUrlResolver.java` | 统一 URL 解析工具，外部 URL → `http://localhost:{port}` 同路径 |

路径：`system-spec/src/main/java/com/xspaceagi/system/spec/utils/FileUrlResolver.java`

```java
public class FileUrlResolver {
    private static final int PORT;
    static {
        String portStr = System.getProperty("server.port", System.getenv().getOrDefault("APP_PORT", "8080"));
        PORT = Integer.parseInt(portStr);
    }
    public static String toAbsoluteUrl(String url) {
        if (url == null) return url;
        if (url.startsWith("/")) {
            return "http://localhost:" + PORT + url;
        }
        try {
            URI uri = new URI(url);
            URI localUri = new URI("http", null, "localhost", PORT, uri.getPath(), uri.getQuery(), null);
            return localUri.toString();
        } catch (Exception e) {
            return url;
        }
    }
}
```

### 不改的场景

| 场景 | 原因 |
|------|------|
| `FileManagementServiceImpl` 存储的 `fileUrl` | 前端知识库模块依赖完整 URL，改为相对路径会导致上传流程中断 |
| 返回前端的模型代理 URL、分享链接等 | 浏览器需要完整 URL |
| 飞书/钉钉回调、沙箱 OAuth 跳转 | 外部服务需要公网可达的 URL |

## 效果

- 前端拿到的文件 URL 仍是完整 URL，知识库上传流程不受影响
- 后端内部解析文档时用 `http://localhost:8080/api/f/xxx` 访问自己，不会超时
- 发布技能打包 zip 时文件下载走 localhost，不会因外部地址不通而卡住 130 秒
- 文件预览/下载前端走同源代理，自动适配任何访问地址

## 踩坑记录

1. **不能改 fileUrl 存储格式为相对路径**：前端知识库模块检测上传响应的 `url` 字段，要求是完整 URL。改为相对路径后前端不会发起后续的文档创建请求，导致 `knowledge_document` 表为空
2. **downloadFileBytes 有 HTTP fallback**：`PublishApplicationServiceImpl` 和 `SkillApplicationServiceImpl` 中的 `downloadFileBytes` 方法先尝试从本地存储读取，失败后 fallback 到 HTTP 请求。如果 fallback 的 URL 是外部地址，后端访问不到会超时 130 秒
3. **知识库有多个 Parser**：除了 `TikaParser`，还有 `WordParser`、`TXTParser`、`JSONParser`、`PdfSegmentationServiceNative` 都会用 `docUrl` 发 HTTP 请求，都需要加 `FileUrlResolver`
