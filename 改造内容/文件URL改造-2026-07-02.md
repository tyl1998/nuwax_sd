# 文件 URL 改造 - 2026-07-02

## 问题背景

通过 IP 地址访问平台时（如 `http://10.94.77.42:18080`），出现两个问题：

1. **CORS 403**：后端 `CORS_ALLOW_ORIGIN` 只配置了 `localhost`，未包含 IP 地址
2. **文件 URL 错误**：后端生成的文件 URL 使用 `siteUrl` 拼接绝对地址，导致：
   - 前端拿到的 URL 指向特定 IP/域名，换网络环境就失效
   - 后端内部解析文档时（TikaParser 等）用该 URL 发 HTTP 请求，访问不到外部地址导致超时

## 改造方案

### 核心思路

- 文件 URL 存储为**相对路径** `/api/f/{fileKey}`，不带域名
- 前端走同源代理，自动适配任何访问地址
- 后端内部通过 `FileUrlResolver` 将相对路径转为 `http://localhost:{port}/api/f/{fileKey}`，自己访问自己

### 新增文件

| 文件 | 说明 |
|------|------|
| `FileUrlResolver.java` | 统一 URL 解析工具，相对路径 → `http://localhost:{port}` + 相对路径 |

路径：`system-spec/src/main/java/com/xspaceagi/system/spec/utils/FileUrlResolver.java`

```java
public class FileUrlResolver {
    private static final String LOCAL_BASE;
    static {
        String port = System.getProperty("server.port", System.getenv().getOrDefault("APP_PORT", "8080"));
        LOCAL_BASE = "http://localhost:" + port;
    }
    public static String toAbsoluteUrl(String url) {
        if (url != null && url.startsWith("/")) {
            return LOCAL_BASE + url;
        }
        return url;
    }
}
```

### 不需要改的 siteUrl 使用场景

以下场景仍然需要 `siteUrl`（外部绝对 URL），不受本次改造影响：

| 场景 | 原因 |
|------|------|
| 返回前端的模型代理 URL、分享链接、临时会话链接 | 浏览器需要完整 URL |
| 飞书/钉钉回调、沙箱 OAuth 跳转 | 外部服务需要公网可达的 URL |
| MCP 导出配置、沙箱环境变量 `PLATFORM_BASE_URL` | 外部集成需要完整 URL |

## 验证

- 新上传文件的 `fileUrl` 字段为相对路径 `/api/f/xxx`
- 知识库文档解析（TikaParser）不再超时
- 前端通过 IP 或 localhost 访问，文件预览和下载均正常
- 已有文件（绝对 URL）仍可正常访问（`FileUrlResolver` 对非 `/` 开头的 URL 原样返回）
