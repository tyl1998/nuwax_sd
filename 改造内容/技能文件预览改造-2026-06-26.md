# 技能文件预览改造 - 2026-06-26

## 1. 文件预览跨域认证问题修复

### 问题描述

技能详情页（`skill-details`）中，新技能的文件通过 `fileProxyUrl` 获取内容。`fileProxyUrl` 格式为 `http://localhost:8080/api/f/...`（绝对路径，指向后端），而前端页面在 `http://localhost`（nginx 代理）。浏览器将不同端口视为不同源，跨域请求默认不携带 Cookie，导致后端返回 `4010 未登录或登录超时`。

同时，`localStorage` 中 `ACCESS_TOKEN` 为空 —— 后端登录接口在非 DEBUG 模式且无 `X-Client-Type` 请求头时，会清空响应体中的 token（`UserController.java:347-348`），token 仅写入 HttpOnly Cookie `ticket`。

### 根因链路

```
前端页面 http://localhost
  → fetchContentFromUrl("http://localhost:8080/api/f/xxx")
    → 原生 fetch，无 credentials 配置
      → 跨域请求不携带 Cookie
        → 后端 AuthInterceptor 读不到 Cookie ticket
          → 返回 4010
```

### 修复方案：绝对 URL 转同源相对路径

将 `http://localhost:8080/api/f/xxx` 转为 `/api/f/xxx`，请求走 nginx 同源代理，浏览器自动携带 Cookie。

### 改动文件

#### 1.1 `src/utils/runtimeConfig.ts` — 新增 `toSameOriginUrl` 工具函数

```typescript
/**
 * 将绝对 URL 转为当前源的相对路径，确保请求走同源代理，避免跨域导致 Cookie 丢失
 * 例: "http://localhost:8080/api/f/xxx" → "/api/f/xxx"
 */
export const toSameOriginUrl = (url: string): string => {
  if (!url) return url;
  if (!isAbsoluteUrl(url)) {
    return withBaseUrl(url);
  }
  try {
    const parsed = new URL(url);
    return parsed.pathname + parsed.search;
  } catch {
    return url;
  }
};
```

#### 1.2 `src/services/skill.ts` — `fetchContentFromUrl` 使用同源 URL

```typescript
// 改前
const fullUrl = isAbsoluteUrl(url) ? url : withBaseUrl(url);

// 改后
const fullUrl = toSameOriginUrl(url);
```

#### 1.3 `src/components/FileTreeView/index.tsx` — `renderContent` 中 fileProxyUrl 同源处理

```typescript
// 改前
if (fileProxyUrl && !/^(https?:)?\/\//i.test(fileProxyUrl)) {
  fileProxyUrl = withBaseUrl(fileProxyUrl);
}

// 改后
if (fileProxyUrl) {
  fileProxyUrl = toSameOriginUrl(fileProxyUrl);
}
```

### 验证

- 新技能（id=10）SKILL.md 预览正常，不再返回 4010
- 新技能 xlsx 文件预览正常

## 2. 文件预览缩放功能

### 问题描述

文件预览页面不支持放大缩小，只有 XMind 通过 `markmap-view` 库内置了 `zoom: true, pan: true`。其他文件类型（docx/xlsx/pdf/pptx/md/html/图片）完全没有缩放能力，内容模糊时无法调整。

### 实现方案

在 `FilePreview` 组件中添加通用缩放控制，适用于所有文件类型。

### 改动文件

#### 2.1 `src/components/business-component/FilePreview/index.tsx`

**新增导入**：`MinusOutlined`、`PlusOutlined`

**新增缩放状态和方法**：

```typescript
const [scale, setScale] = useState(1);
const SCALE_MIN = 0.25;
const SCALE_MAX = 3;
const SCALE_STEP = 0.15;

const handleZoomIn = useCallback(() => {
  setScale((prev) => Math.min(prev + SCALE_STEP, SCALE_MAX));
}, []);

const handleZoomOut = useCallback(() => {
  setScale((prev) => Math.max(prev - SCALE_STEP, SCALE_MIN));
}, []);

const handleZoomReset = useCallback(() => {
  setScale(1);
}, []);
```

**Ctrl+滚轮缩放**：

```typescript
useEffect(() => {
  const container = containerRef.current?.parentElement;
  if (!container) return;
  const handleWheel = (e: WheelEvent) => {
    if (e.ctrlKey || e.metaKey) {
      e.preventDefault();
      const delta = e.deltaY > 0 ? -SCALE_STEP : SCALE_STEP;
      setScale((prev) => Math.max(SCALE_MIN, Math.min(SCALE_MAX, prev + delta)));
    }
  };
  container.addEventListener('wheel', handleWheel, { passive: false });
  return () => container.removeEventListener('wheel', handleWheel);
}, []);
```

**键盘快捷键**：Ctrl+加号放大、Ctrl+减号缩小、Ctrl+0 重置

**工具栏**：始终显示缩放控件（缩小按钮 / 百分比标签 / 放大按钮），百分比标签点击重置为 100%

**内容区域缩放**：
- 非文档类型（image/markdown/html/text）：用 `zoomWrapper` 包裹，应用 `transform: scale()`
- 文档类型（docx/xlsx/pdf/pptx/xmind）：在 `previewContent` 容器上直接应用 `transform: scale()`

## 4. 登录认证机制说明

### 认证链路

1. 登录接口 `/api/user/passwordLogin` → 后端生成 JWT token
2. token 写入 HttpOnly Cookie `ticket`（`UserController.java:337-341`）
3. 响应体中 token 被清空（`UserController.java:347-348`）：`X-Client-Type` 为空且非 DEBUG 模式时 `token = ""`
4. 前端 `localStorage.setItem('ACCESS_TOKEN', token)` 存入空值

### 请求认证

`AuthInterceptor.java:182-197`：
1. 优先读 Cookie `ticket`
2. 其次读 `Authorization: Bearer <token>` 头

### 同源请求

- 前端页面 `http://localhost` → nginx 代理 → 后端 `http://localhost:8080`
- Cookie `ticket` 由 `http://localhost` 设置，同源请求自动携带
- 其他 API（如 `/api/skill/10`）走同源，认证正常

### 跨域请求（已修复）

- `fileProxyUrl` 指向 `http://localhost:8080`，跨域请求不带 Cookie
- 修复：`toSameOriginUrl()` 将绝对 URL 转为相对路径，走同源代理
