# 智能体页面绑定机制说明

## 1. 什么是页面绑定

页面绑定是将你在「网页应用开发」（SpacePageDevelop）中开发的页面，作为一个组件绑定到智能体上。绑定后，智能体在对话过程中可以根据需要打开该页面并传入动态参数，页面内容在聊天气泡旁的 iframe 中展示。

---

## 2. 完整流程

### 阶段一：页面开发

在「网页应用开发」中创建页面项目（React/Vue3）：

```
SpacePageDevelop
  ├─ 创建项目（上传 ZIP / 在线模板 / 反向代理）
  ├─ 配置路由：如 /view、/edit、/detail
  │   └─ 每个路由定义输入参数 args：
  │       └─ BindConfigWithSub { name, dataType, require, default, inputType(Query|Path) }
  └─ 发布页面 → 构建部署，生成 basePath（如 /page/381234-9923/）
```

### 阶段二：绑定到智能体

在智能体编辑 → 编排配置中：

```
AgentArrangeConfig
  ├─ "界面" 区域 → 点击 "添加页面"
  │   └─ 从已发布的页面列表中选择
  ├─ 页面设置弹窗（PageSettingModal）：
  │   ├─ VisibleToLLM — 页面是否对模型可见（LLM 能否自行决定打开）
  │   ├─ HomeIndex — 是否设为默认首页（智能体打开时自动展示）
  │   └─ 自定义名称/图标
  └─ 可选：在开场白引导问题或事件绑定中引用页面路由
```

### 阶段三：运行时调用

```
用户发送消息
  └─ SSE 连接 /api/agent/conversation/chat
       └─ PROCESSING 事件 type='Page'
            └─ result.input = {
                 uri: "/page/381234-9923/view",
                 arguments: { id: "123", name: "测试" },  // ← 动态参数
                 method: "browser_open_page",
                 data_type: "html",
                 uri_type: "Page"  // 或 "Link"
               }
                   ↓
         showPagePreview({ uri, params: arguments, executeId })
                   ↓
         PagePreviewIframe → <iframe src="http://host/page/381234-9923/view?id=123&name=测试" />
```

## 3. 动态数据如何传入页面

### 3.1 参数来源

页面打开时携带的 `arguments`（动态参数）可以来自：

| 来源 | 说明 |
|------|------|
| **LLM 自主生成** | VisibleToLLM=Yes 时，LLM 根据对话上下文自行决定参数值 |
| **开场白引导问题** | 预填的参数值 + 系统变量（`{{AGENT_ID}}`、`{{USER_ID}}`、`{{USER_NAME}}`、`{{USER_UID}}`） |
| **事件绑定配置** | 事件触发时传入绑定好的参数 |
| **用户手动点击** | 聊天气泡旁的 "眼睛" 图标，点开后重新执行页面展示 |

### 3.2 参数绑定方式

每个页面路由的参数定义在 `pathArgConfigs` 中：

```typescript
{
  pageUri: "/view",
  name: "详情页面",
  args: [
    { name: "id", dataType: "Integer", require: true, inputType: "Query" },
    { name: "name", dataType: "String", require: false, inputType: "Query", bindValue: "默认名称" }
  ]
}
```

- `inputType: Query` — 参数拼接到 URL query string（`?id=123&name=测试`）
- `inputType: Path` — 参数替换到 URL 路径中（`/view/123`）
- `bindValue` — 默认值
- `require` — 是否必填

### 3.3 页面如何消费参数

页面作为独立的前端应用运行在 iframe 中，通过标准 Web 方式获取参数：

```javascript
// 页面内代码（React/Vue3 应用）
const params = new URLSearchParams(window.location.search);
const id = params.get('id');    // "123"
const name = params.get('name'); // "测试"

// 然后根据 id 发起 API 请求渲染内容
fetch(`/api/data/${id}`).then(...)
```

## 4. 三种打开方式

### 方式一：LLM 自主打开（最常用）

配置 `visibleToLLM = Yes`。LLM 在对话中判断需要展示页面时，通过 tool call 返回 `type='Page'` 的 PROCESSING 事件，前端自动打开 iframe 预览。

### 方式二：引导问题点击

开场白中的"扩展页面路径"类引导问题，用户点击后直接打开指定页面路径。

### 方式三：事件触发

LLM 在输出文本中嵌入带有 `class="event"` 的 HTML 标签，`useMessageEventDelegate` 拦截点击事件，查找对应的事件配置并打开页面。

## 5. 重要概念

| 概念 | 说明 |
|------|------|
| `basePath` | 页面部署后的基础路径，如 `/page/381234-9923/` |
| `pageUri` | 页面内路由，如 `/view`、`/edit` |
| 完整 URL | `basePath + pageUri + ?queryParams` |
| `PageApp` vs `Page` | `PageApp` 是纯页面型的智能体（无聊天）；`Page` 是绑定到聊天型智能体上的页面组件 |
| `PROCESSING` 事件 | SSE 事件类型，标记工具/页面正在执行 |
| `browser_open_page` | 打开新页面 |
| `browser_navigate_page` | 页面内导航，并支持将 HTML 内容报告给 LLM（`data_type=html`） |
