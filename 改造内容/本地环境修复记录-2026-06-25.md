# 本地环境修复记录 - 2026-06-25

## 1. agent-runner-image-puller 优化（docker-compose.yml）

**文件**：`nuwax_computer_deploy/docker-computer/docker-compose.yml`

**改动**：`agent-runner-image-puller` 服务新增 `FORCE_PULL` 环境变量，默认跳过已有镜像

```yaml
environment:
  # 是否强制拉取最新镜像：true=每次都拉取最新，false/不设置=本地已有则跳过
  - FORCE_PULL=${FORCE_PULL:-false}
```

**使用方式**：
```bash
# 默认模式（跳过已有镜像）
docker compose up -d

# 强制拉取最新镜像
FORCE_PULL=true docker compose up -d
```

## 2. rcoder nginx 配置修复（sub_app_multi_apps.conf）

**文件**：`nuwax_computer_deploy/docker-computer/config/rcoder/sub_app_multi_apps.conf`

**改动**：添加 `/computer/` 和 `/api/` 代理规则到 rcoder 内部 nginx

```nginx
# Computer 智能体桌面路由（VNC、音频、IME 等）代理到 rcoder 后端
location /computer/ {
    proxy_pass http://127.0.0.1:8086;
    proxy_http_version 1.1;
    proxy_set_header Upgrade $http_upgrade;
    proxy_set_header Connection "upgrade";
    proxy_set_header Host $host;
    proxy_set_header X-Real-IP $remote_addr;
    proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
    proxy_set_header X-Forwarded-Proto $scheme;
    proxy_read_timeout 3600s;
    proxy_send_timeout 3600s;
}

# API 路由代理到 rcoder 后端
location /api/ {
    proxy_pass http://127.0.0.1:8086;
    proxy_http_version 1.1;
    proxy_set_header Host $host;
    proxy_set_header X-Real-IP $remote_addr;
    proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
    proxy_set_header X-Forwarded-Proto $scheme;
}
```

**说明**：rcoder 内部 nginx 原本只有 `/page/` 和 `/` 路由，`/computer/` 请求会 404。此改动让 rcoder 内部 nginx 也能正确代理 VNC 桌面请求。

## 3. 前端 nginx 配置修复（容器内修改，需持久化）

**问题**：前端容器内的 nginx 配置没有 `/computer/` 路由，浏览器访问 VNC 桌面会走 SPA fallback 返回 404。

**改动**：

### 3.1 主 nginx.conf 添加 WebSocket map 块

在 `/etc/nginx/nginx.conf` 的 `http {` 后添加：

```nginx
map $http_upgrade $connection_upgrade {
    default upgrade;
    '' close;
}
```

### 3.2 default.conf 添加 /computer/ 路由

在 `location / {` 之前添加：

```nginx
# Computer 智能体桌面路由 - 代理到后端 ComputerProxy (18085)
location /computer/ {
    proxy_pass http://nuwax-backend:18085;
    proxy_http_version 1.1;
    proxy_set_header Upgrade $http_upgrade;
    proxy_set_header Connection $connection_upgrade;
    proxy_set_header Host $host;
    proxy_set_header X-Real-IP $remote_addr;
    proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
    proxy_set_header X-Forwarded-Proto $scheme;
    proxy_buffering off;
    proxy_read_timeout 3600s;
    proxy_send_timeout 3600s;
}
```

### 3.3 网络配置

```bash
# 创建自定义网络，让前端和后端能通过容器名互访
docker network create nuwax-net
docker network connect nuwax-net nuwax-frontend
docker network connect nuwax-net nuwax-backend
```

**链路**：
```
浏览器 → 前端 nginx (80) → /computer/* → 后端 ComputerProxy (18085) → rcoder
```

**⚠️ 注意**：前端容器内的修改在容器重启后会丢失。需要持久化有两种方式：
- 重建前端镜像
- 将 nginx 配置文件挂载到宿主机（类似 `nuwax_deploy` 的方式）

备份文件已保存到：`nuwax_deploy/docker/config/nginx.conf.frontend-backup`
