# 基础服务拆分启动

## 1. 目标

将原 `nuwax_deploy/docker/docker-compose.yml` 中的基础服务单独拆分启动，供改造后的前端 `nuwax` 与后端 `nuwax-backend` 独立容器复用。

拆分服务包括：

- MySQL
- Redis
- Milvus
- MinIO
- Quickwit
- Elasticsearch

## 2. 真实落点

真实改动目录：

```bash
/Users/atan/Desktop/work/vs_code_nuwax/nuwax_deploy/docker
```

新增文件：

- `docker-compose-infra.yml`
- `start-infra-containers.sh`

## 3. 数据挂载原则

为避免数据丢失，基础服务继续复用原 compose 的本地目录挂载，不切换到新的 Docker volume。

关键挂载目录：

- MySQL：`./data/mysql`、`./logs/mysql`、`./config/mysql.cnf`
- Redis：`./data/redis`、`./logs/redis`、`./config/redis.conf`
- Milvus：`./data/milvus/data`、`./data/milvus/etcd`、`./logs/milvus`、`./config/milvus/*`
- MinIO：`./data/minio`
- Quickwit：`./data/quickwit`、`./logs/quickwit`
- Elasticsearch：`./data/elasticsearch`、`./logs/elasticsearch`、`./config/elasticsearch/*`

## 4. 启动命令

推荐使用单容器脚本启动：

```bash
cd /Users/atan/Desktop/work/vs_code_nuwax/nuwax_deploy/docker
./start-infra-containers.sh
```

该脚本内部使用 `docker run` 分别启动 MySQL、Redis、Milvus、MinIO、Elasticsearch，不再使用 compose 编排基础服务，也不再加入 `docker_agent-network`。

如需单独查看脚本中的命令：

```bash
sed -n '1,260p' /Users/atan/Desktop/work/vs_code_nuwax/nuwax_deploy/docker/start-infra-containers.sh
```

## 5. 查看状态

```bash

```

## 7. 后端连接参数

改造后的 `nuwax-backend` 单容器可使用以下默认外部依赖地址：

```bash
MYSQL_HOST=host.docker.internal
MYSQL_PORT=13306
MYSQL_DATABASE=agent_platform
MYSQL_USER=agent_platform
MYSQL_PASSWORD=admin123
REDIS_HOST=host.docker.internal
REDIS_PORT=16379
REDIS_PASSWORD=123456
MILVUS_URI=http://host.docker.internal:19530
ES_URL=http://host.docker.internal:9200
ES_USERNAME=elastic
ES_PASSWORD=elastic123
```

不推荐后端依赖 `docker_agent-network` 或容器别名；跨机器模式统一使用 IP、域名或宿主机映射端口。

## 8. 前端与后端网络配置

前端 `nuwax-frontend` 通过 `/computer/` 路由代理 VNC 桌面请求到后端 ComputerProxy（18085 端口）。前后端需要在同一个 Docker 网络中才能通过容器名互访。

```bash
# 创建自定义网络
docker network create nuwax-net 2>/dev/null || true

# 连接前后端容器
docker network connect nuwax-net nuwax-frontend
docker network connect nuwax-net nuwax-backend
```

后端启动时需暴露 18085 端口（ComputerProxy）：

```bash
-p 18085:18085
```

前端启动时需设置 `COMPUTER_PROXY_URL` 环境变量：

```bash
-e COMPUTER_PROXY_URL=http://nuwax-backend:18085
```

详细启动命令参见 [`基础服务Jenkins单容器启动命令.md`](基础服务Jenkins单容器启动命令.md) 第 9、10 节。

## 8. 注意事项

- 不执行 `docker compose down -v`，避免删除 volume 或误清数据。
- 不删除 `./data/*`、`./logs/*`、`./config/*` 目录。
- `start-infra-containers.sh` 只删除并重建容器本体，所有数据仍在原 bind mount 目录中。
- 如原整套 compose 仍在运行同名服务或占用端口，需先停旧基础服务容器，再启动拆分后的基础服务。
