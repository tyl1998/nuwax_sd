## 3. 推荐阅读顺序

### 第一步：理解注册与路由（2小时）

1. `main.rs` — 了解服务器如何启动，`get_router()` 挂载了哪些路由
2. `model/mcp_config.rs` — 搞清楚 `McpConfig`、`McpType`、`McpProtocol` 的关系
3. `handlers/mcp_add_handler.rs` — 注册 API 的完整流程
4. `server/task/mcp_start_task.rs` → `integrate_server_with_axum()` — 后端启动与路由注册

### 第二步：理解动态路由（2小时）

5. `server/mcp_dynamic_router_service.rs` — 核心文件，逐段读完
   - `call()` 方法：路由查找 → 状态检查 → 启动锁 → 健康检查 → 转发/重启
   - `get_route()` / `register_route()` — DashMap 操作
   - `start_mcp_and_handle_request()` — 重启后透传

### 第三步：理解生命周期（1小时）

6. `server/task/schedule_check_mcp_live.rs` — 定时巡检逻辑
   - Persistent：子进程存活检测
   - OneShot：空闲超时 + 探测失败计数
   - `restart_mcp_service()` — 冷却期 + 重启流程

### 第四步：了解附属服务（按需）

7. `document-parser/src/routes.rs` — 接口全览
8. `fastembed/src/server/mod.rs` — Embedding 服务路由
9. `voice-cli/src/main.rs` — 语音服务入口

## 4. 与 nuwax-backend 对接点

如果你从 `nuwax-backend` 侧切入，追踪 MCP 工具调用链：

```
nuwax-backend
└── agent/infra/rpc/McpRpcService.java
    ├── registerMcp()   → POST /mcp/sse/add
    ├── checkStatus()   → GET  /mcp/sse/{id}/check_status
    └── 实际 MCP 调用  → Spring AI McpClient
                          → GET  /mcp/sse/{id}/sse（SSE 连接）
                          → POST /mcp/sse/{id}/message（发消息）
```

配置 JSON（`mcp_json_config`）由 nuwax-backend 的 MCP 插件配置页面构造，格式遵循 MCP 标准 `mcpServers` 结构。

## 5. 常见问题定位

| 现象 | 检查位置 |
|------|---------|
| 注册返回 400 | `mcp_add_handler.rs` 的 `McpRouterPath::from_uri_prefix_protocol` |
| 调用返回 503 | `mcp_dynamic_router_service.rs`：Pending 状态或启动锁被占用 |
| 调用返回 0006 | OneShot 服务已退出，需重新注册 |
| 调用返回 0004 | Persistent 不健康且找不到配置，排查配置缓存写入 |
| 自动重启不生效 | 检查 `GLOBAL_RESTART_TRACKER.can_restart()`，是否在 30s 冷却期内 |
| 后端协议检测失败 | `mcp_start_task.rs` 的 `detect_mcp_protocol_with_headers()`，检查 URL 可达性和 auth_token |
