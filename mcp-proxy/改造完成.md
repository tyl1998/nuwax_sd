### 1. Dockerfile 端口对齐

- **文件**：`docker/Dockerfile.mcp-proxy`

- **文件**：`mcp-proxy/src/config.rs`
- **已有支持**：

### 4. 构建与部署脚本

- **文件**：`scripts/build-runtime-image.sh` — Jenkins/本地统一构建镜像
- **文件**：`scripts/deploy-runtime-container.sh` — 单容器启动脚本

## Docker 启动命令

### 使用说明

部署脚本 `scripts/deploy-runtime-container.sh` 已内置所有默认参数，**直接运行即可，无需传任何变量**。脚本会自动：

1. 创建日志和缓存目录
2. 停止并删除同名旧容器
3. 启动新容器
4. 等待健康检查通过

### 构建镜像

```bash
cd /Users/atan/Desktop/work/vs_code_nuwax/mcp-proxy

# 使用构建脚本（推荐）
bash scripts/build-runtime-image.sh
```

构建脚本内置默认参数：

| 变量 | 默认值 | 说明 |
|------|--------|------|
| `IMAGE_NAME` | `mcp-proxy` | 镜像名 |
| `IMAGE_TAG` | `local` | 镜像 tag |

### 启动容器

```bash
cd /Users/atan/Desktop/work/vs_code_nuwax/mcp-proxy

# 直接启动（使用默认参数，无需传变量）
bash scripts/deploy-runtime-container.sh
```

部署脚本内置默认参数：

| 变量 | 默认值 | 说明 |
|------|--------|------|
| `IMAGE_NAME` | `mcp-proxy` | 镜像名 |
| `IMAGE_TAG` | `local` | 镜像 tag |
| `CONTAINER_NAME` | `nuwax-mcp-proxy` | 容器名 |
| `HOST_PORT` | `8020` | 宿主机端口 |
| `MCP_PROXY_PORT` | `8089` | 容器内端口 |
| `CONFIG_FILE` | `nuwax_deploy/docker/config/mcp_config.yml` | 配置文件路径 |
| `LOG_DIR` | `nuwax_deploy/docker/logs/mcp_proxy` | 日志目录 |
| `UV_CACHE_DIR` | `nuwax_deploy/docker/data/uv_cache/uv` | uv 包缓存 |
| `NPM_CACHE_DIR` | `nuwax_deploy/docker/data/npx_cache/.npm` | npm 包缓存 |
| `MCP_PROXY_LOG_LEVEL` | `info` | 日志级别 |
| `RUST_LOG` | `info` | Rust 日志级别 |

脚本实际执行的 `docker run` 命令：

```bash
docker run -d \
  --name nuwax-mcp-proxy \
  --restart=always \
  -p 8020:8089 \
  -e MCP_PROXY_PORT=8089 \
  -e MCP_PROXY_LOG_DIR=/app/logs \
  -e MCP_PROXY_LOG_LEVEL=info \
  -e RUST_LOG=info \
  -v nuwax_deploy/docker/config/mcp_config.yml:/app/config.yml:ro \
  -v nuwax_deploy/docker/logs/mcp_proxy:/app/logs \
  -v nuwax_deploy/docker/data/uv_cache/uv:/root/.cache/uv \
  -v nuwax_deploy/docker/data/npx_cache/.npm:/root/.npm \
  mcp-proxy:local
```

### Jenkins 启动命令

## 后续注意事项

1. **镜像较大**：runtime 阶段包含 Rust、Node、Python、Deno、uv，镜像较大，但有利于兼容动态 MCP 工具
2. **缓存目录**：uv 和 npm 缓存目录必须保留挂载，否则首次工具调用会变慢
3. **Deno/uv 安装**：依赖外网脚本，Jenkins 弱网环境可能失败，后续可沉淀内部基础镜像
4. **不执行 `docker compose down -v`**：不删除原 `data/uv_cache`、`data/npx_cache`、`logs/mcp_proxy` 目录
