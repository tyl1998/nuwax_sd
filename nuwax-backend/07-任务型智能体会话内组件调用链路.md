# 任务型智能体会话内组件调用链路

## 1. 这篇文档解决什么问题

梳理任务型智能体（TaskAgent）在一次会话中如何调用插件、工作流、知识库（文本）、数据表、MCP、长期记忆这六大类组件，包括各自的触发时机、执行路径和代码落点。

## 2. 总体执行架构

核心入口是 `AgentExecutor.execute()`，按以下顺序编排：

```
execute()
  ├─ 1. 加载上下文记忆 (contextMessages)
  ├─ 2. 查询长期记忆 (longMemory)
  ├─ 3. 变量识别与设置 (variableSet) → 模型提取变量
  └─ 4. doNext()
       ├─ 4a. invokeAndRemoveAutoComponents()  ← AUTO 组件在此执行
       ├─ 4b. buildSystemPrompt()              ← 组装 system prompt
       ├─ 4c. convertToComponentConfigList()    ← 构建 Function Calling 工具列表
       └─ 4d. 分支执行
            ├─ TaskAgent → sandboxAgentClient.chat()  ← 走沙箱容器
            └─ Default   → modelInvoker.invoke()       ← 走后端直接模型调用
```

### 2.1 关键代码文件

| 文件 | 位置 | 职责 |
|------|------|------|
| `AgentExecutor.java` | `app-platform-agent-core-infra/.../component/agent/` | 主执行引擎：上下文加载 → 组件编排 → 模型/沙箱调用 → 流式推流 |
| `ModelInvoker.java` | `app-platform-agent-core-infra/.../component/model/` | Default Agent 的模型调用（Function Calling 循环） |
| `SandboxAgentClient.java` | `app-platform-agent-core-infra/.../component/agent/` | TaskAgent 走 rcoder 沙箱的客户端 |
| `ConversationApplicationServiceImpl.java` | `app-platform-agent-core-application/.../service/` | 会话入口、组件调用方式转换、记忆查询 |
| `PluginExecutor.java` | `app-platform-agent-core-infra/.../component/plugin/` | 插件执行器 |
| `WorkflowExecutor.java` | `app-platform-agent-core-infra/.../component/workflow/` | 工作流执行器 |
| `KnowledgeBaseSearcher.java` | `app-platform-agent-core-infra/.../component/knowledge/` | 知识库检索（向量 + 全文 + QA） |
| `McpExecutor.java` | `app-platform-agent-core-infra/.../component/mcp/` | MCP 工具调用 |
| `TableExecutor.java` | `app-platform-agent-core-infra/.../component/table/` | 数据表执行器 |

### 2.2 调用方式枚举

前端定义了 4 种调用方式（`nuwax/src/types/enums/agent.ts:72-82`）：

| 枚举值 | 含义 | 转换规则 |
|--------|------|---------|
| `AUTO` | 自动调用（每轮对话都执行） | 直接在 `invokeAndRemoveAutoComponents()` 中执行 |
| `ON_DEMAND` | 按需调用（模型根据任务决定） | 注册为 Function Calling 工具 |
| `MANUAL` | 手动选择 | 未选中则移除，选中则改为 AUTO |
| `MANUAL_ON_DEMAND` | 手动选择 + 按需调用 | 未选中则移除，选中则改为 ON_DEMAND |

## 3. 插件 (Plugin)

插件有两条调用路径，取决于 `invokeType`。

### 3.1 AUTO（自动调用）

- **位置**：`AgentExecutor.invokeAndRemoveAutoComponents()` (`:682-701`)
- **逻辑**：遍历组件列表，发现 `PluginInvokeTypeEnum.AUTO` → 直接执行
- **调用链**：

```
invokePlugin() (:867-907)
  → PluginContext.builder()...build()
  → pluginExecutor.execute(pluginContext).block()
  → 返回 PluginExecuteResultDto
  → 结果拼入 autoToolCallResult，作为后续模型的参考信息
```

### 3.2 ON_DEMAND（按需调用）

- **位置**：`ModelInvoker.buildFunctionCallbacks()` (`:743-823`)
- **逻辑**：插件被转为 `FunctionToolCallback`，注册为模型可用工具
- **执行链**：

```
模型返回 tool_call
  → componentExecute() (:826-907)
    → executePlugin() (:1056-1082)
      → pluginExecutor.execute(pluginContext).timeout(180s).block()
      → 返回结果给模型
```

- **特点**：模型自行决定何时调用，支持多轮 Function Calling 循环

### 3.3 MANUAL → 被前端选中后转为 AUTO

- **位置**：`ConversationApplicationServiceImpl` (`:1183-1200`)
- **逻辑**：`MANUAL` 未选中则移除，选中改为 `AUTO`；`MANUAL_ON_DEMAND` 未选中则移除，选中改为 `ON_DEMAND`

---

## 4. 工作流 (Workflow)

与插件结构完全相同的两条路径。

### 4.1 AUTO（自动调用）

- **位置**：`invokeAndRemoveAutoComponents()` (`:722-751`)
- **调用链**：

```
invokeWorkflow() (:824-865)
  → WorkflowContext.builder()...build()
  → workflowExecutor.execute(workflowContext).block()
  → 返回结果
```

- **特殊能力**：支持 `directOutput`（直接输出模式）— 工作流结果直接作为最终回复，不再经过模型

### 4.2 ON_DEMAND（按需调用）

- **位置**：`ModelInvoker.buildFunctionCallbacks()` → `componentExecute()` → `executeWorkflow()` (`:1023-1054`)
- **调用链**：

```
模型返回 tool_call
  → componentExecute()
    → executeWorkflow()
      → workflowExecutor.execute(workflowContext).block()
      → 返回结果给模型
```

- **特殊能力**：同空间时记录节点级执行日志 (`innerExecuteInfo`)

---

## 5. 知识库 (Knowledge)

知识库是"文本"能力的核心载体。

### 5.1 AUTO（自动调用）

- **位置**：`invokeAndRemoveAutoComponents()` (`:597-665`)
- **调用链**：

```
queryKnowledgeBase() (:925-952)
  → SearchContext.builder()...build()
  → knowledgeBaseSearcher.search(searchContext).block()
  → 三路检索合并：向量 (Milvus) + 全文 (Elasticsearch) + QA
  → 结果格式化为 Markdown 拼入 autoToolCallResult
```

### 5.2 ON_DEMAND（按需调用）

- **位置**：`ModelInvoker.buildFunctionCallbacks()` → `componentExecute()` → `queryKnowledge()` (`:1002-1021`)
- **工具参数**：`{ query: "搜索关键词" }`
- **超时**：60 秒

### 5.3 检索策略

| 检索路径 | 实现 | 特点 |
|---------|------|------|
| **向量检索** | Milvus ANN 搜索 | 语义相似，能找到意思相近但用词不同的段落 |
| **全文检索** | Elasticsearch BM25 | 精确关键词匹配，对专有名词效果好 |
| **QA 检索** | 预先生成 QA 对后向量化 | 问题-答案对齐，对问答型知识库效果最好 |

---

## 6. 技能 (Skill)

技能 **不是可调用工具**，不进入 `convertToComponentConfigList()`，也不进入 `buildFunctionCallbacks()`。

### 6.1 注入机制

- **位置**：`ConversationApplicationServiceImpl` (`:1234-1243`)
- **逻辑**：

```java
// 技能名称被注入到 userPrompt
userPromptBuilder.append("\nPlease use the following skills to complete user tasks. ");
userSelectedSkills.forEach(skillConfigDto -> {
    userPromptBuilder.append("- ").append(skillName).append("\n");
});
```

- **执行位置**：
  - TaskAgent：Agent 引擎从工作空间 `skills/` 目录读取 SKILL.md 文件自行决策
  - Default：模型根据提示词自行决定是否参考技能

### 6.2 与可调用组件的关键区别

| 维度 | 技能 (Skill) | 插件/工作流/MCP/数据表 |
|------|-------------|----------------------|
| 本质 | 文件资源（SKILL.md + 代码） | API/函数调用 |
| 调用方式 | 模型自行读取文件 | Function Calling 工具调用 |
| 自动调用 (AUTO) | 不支持 | 支持 |
| 执行位置 | Agent 引擎内部 | nuwax-backend 后端 |

---

## 7. 数据表 (Table)

数据表 **不走 AUTO 路径**，全部作为 Function Calling 工具注册。

### 7.1 工具注册

- **位置**：`AgentExecutor.convertToComponentConfigList()` (`:1078-1086`)
- **注册方式**：一个数据表注册为 **两个工具**：

| 工具名 | 功能 | 参数 |
|--------|------|------|
| `{表名}插入数据` | INSERT | 从表结构自动生成字段参数 |
| `{表名}SQL查询` | SQL 执行 | `{ sql: "SELECT ..." }`，描述中包含表结构信息 |

### 7.2 执行链

- **位置**：`ModelInvoker.componentExecute()` → `queryTable()` (`:961-1000`)
- **调用链**：

```
模型返回 tool_call
  → componentExecute()
    → queryTable()
      ├── TABLE_DATA_INSERT → TableExecutor.generateInsertSql() → tableExecutor.execute()
      └── TABLE_SQL_EXECUTE → tableExecutor.execute(sql)
```

- **系统变量注入**：执行时自动注入 `uid`, `nick_name`, `user_name`, `agent_id`, `agent_name`

---

## 8. MCP

MCP 有两条路径，与插件/工作流类似。

### 8.1 AUTO（自动调用）

- **位置**：`invokeAndRemoveAutoComponents()` (`:702-721`)
- **调用链**：

```
invokeMcp() (:795-822)
  → McpContext.builder()...build()
  → mcpExecutor.execute(mcpContext).blockLast()
  → 返回 McpExecuteOutput
```

### 8.2 ON_DEMAND（按需调用）

- **位置**：`ModelInvoker.buildFunctionCallbacks()` → `componentExecute()` → `executeMcp()` (`:943-959`)
- **特殊**：MCP 工具的 inputSchema 从 `mcpDto.getDeployedConfig().getTools()` 中获取，不从 `inputArgs` 构建

### 8.3 TaskAgent 特殊处理

- **位置**：`ConversationApplicationServiceImpl` (`:1246-1281`)
- **逻辑**：组件类型的 MCP（`InstallTypeEnum.COMPONENT`）、Plugin、Knowledge、Table、Workflow 会被转换为 **代理 MCP 服务**
- **调用链**：

```
agentApplicationService.buildProxyMcp(agentConfigDto, isDev)
  → mcpRpcService.getExportMcpServerConfig(userId, proxyMcpId, userAccessKeyConfig)
  → agentConfigDto.setProxyMcpServerConfig(exportMcpServerConfig)
```

- **效果**：沙箱内的 Agent 引擎通过 MCP 协议直接调用这些组件

---

## 9. 长期记忆 (Memory)

### 9.1 读取（对话开始前）

- **位置**：`AgentExecutor.execute()` (`:153-162`)
- **条件**：`openLongMemory == Open`
- **调用链**：

```
conversationApplicationService.queryMemory(
    tenantId, userId, agentId, message, "", justKeywordMatch, filterSensitive
)
  → Milvus 向量检索，12 大类 50+ 子类
  → 返回记忆文本
```

- **注入方式**：结果写入 `agentContext.longMemory`，在 `buildUserMessage()` (`:513-518`) 中以 `<user-memory>` 标签包裹注入 user prompt

```java
stringBuilder.append("\n<user-memory>\n")
    .append(JSON.toJSONString(agentContext.getLongMemory()))
    .append("\n</user-memory>\n");
```

### 9.2 写入（对话结束后）

- **位置**：`AgentExecutor.chatComplete()` (`:440-453`)
- **条件**：同上 + conversationId 不以 `agent:` 开头（排除子 Agent 调用）
- **调用链**：

```
conversationApplicationService.pushToSummaryQueue(conversationId)
  → 异步任务推入记忆摘要队列
  → 记忆提取模型从对话中识别用户信息
  → 去重 / 合并旧记忆
  → 写入 Milvus 按类别向量化存储
```

### 9.3 记忆分类体系

| 分类 | 示例子分类 |
|------|---------|
| 个人信息 | 姓名、年龄、职业、所在城市 |
| 工作信息 | 公司、职位、主要工作内容 |
| 兴趣爱好 | 运动偏好、阅读习惯、饮食偏好 |
| 技术背景 | 编程语言、技术栈、开发经验 |
| 沟通偏好 | 回答语气、详细程度偏好 |
| 项目/任务 | 正在做的项目、遇到的问题 |

## 10. TaskAgent vs Default Agent 核心区别

| 维度 | TaskAgent（任务型） | Default（默认引擎） |
|------|-------------------|-------------------|
| **执行入口** | `sandboxAgentClient.chat()` | `modelInvoker.invoke()` |
| **组件调用** | 组件转代理 MCP 配置，由沙箱内引擎自行调用 | 后端通过 Spring AI Function Calling 编排 |
| **技能调用** | Agent 引擎从工作空间读取 SKILL.md | 通过 userPrompt 注入技能名称 |
| **上下文轮数** | 固定 10 轮 | 用户可配置 |
| **模型调用** | 沙箱内引擎调用（claude-code-acp-ts / nuwaxcode） | 后端直接调用模型 API |
| **MCP/插件/工作流/知识库/数据表** | 转代理 MCP 服务，容器内通过 MCP 协议调用 | 后端 `componentExecute()` 统一执行 |

### 10.1 TaskAgent 的代理 MCP 转换流程

```
ConversationApplicationServiceImpl (:1246-1281)
  ├─ 筛选组件：Mcp(COMPONENT) + Plugin + Knowledge + Table + Workflow
  ├─ agentApplicationService.buildProxyMcp(agentConfigDto, isDev)
  │    → 构建代理 MCP 服务
  ├─ mcpRpcService.getExportMcpServerConfig(...)
  │    → 导出 MCP Server 配置（含工具定义、认证信息）
  └─ agentConfigDto.setProxyMcpServerConfig(exportMcpServerConfig)
       → 配置注入请求体，沙箱内引擎启动时加载
```

## 11. 调用路径总结表

| 组件 | AUTO（每轮自动执行） | ON_DEMAND（模型 Function Calling） | MANUAL | TaskAgent 特殊处理 |
|------|:---:|:---:|:---:|:---:|
| **插件** | ✅ `pluginExecutor.execute()` | ✅ 注册为 Tool | ✅ 选中后转 AUTO | 转代理 MCP |
| **工作流** | ✅ `workflowExecutor.execute()` | ✅ 注册为 Tool | ✅ 选中后转 AUTO | 转代理 MCP |
| **知识库** | ✅ `knowledgeBaseSearcher.search()` | ✅ 注册为 Tool | ✅ 选中后转 AUTO | 转代理 MCP |
| **数据表** | ❌ | ✅ 注册为 2 个 Tool（Insert + SQL） | ❌ | 转代理 MCP |
| **MCP** | ✅ `mcpExecutor.execute()` | ✅ 注册为 Tool | ✅ 选中后转 AUTO | 代理 MCP 配置注入 |
| **技能** | ❌ | ❌（名称注入 prompt） | ✅ MANUAL_ON_DEMAND | Agent 引擎读取文件 |
| **长期记忆** | ✅ 对话前查询 + 对话后异步摘要 | — | — | 同 Default |

---

## 12. 最终 Prompt 结构

传给模型的 system prompt 大致是：

```
{租户全局 system prompt}

{agent 自定义 system prompt}

{用户长期记忆（若有）}
  - 用户职业：前端工程师
  - 用户偏好：回答简洁，给代码示例

{知识库检索结果（若有，AUTO 模式）}
  ## 参考资料
  [1] 文档名 - 段落内容...

{用户变量（若有）}
  用户姓名：张三

{页面组件信息（若有）}
  ## CUSTOM PAGE
  1. 页面名-描述
     URI:/api/...
     ParameterJsonSchema:...
```

user prompt 大致是：

```
<user-memory>长期记忆 JSON</user-memory>
<user-prompt>用户自定义提示词</user-prompt>
<attachment>附件信息</attachment>
<user-message>用户实际消息</user-message>
```

模型只看到组合后的 prompt，不感知"这段来自知识库，那段来自记忆"。
