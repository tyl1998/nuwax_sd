# 阅读路径

## 1. 按关注点快速定位

### 想搞清楚"前后端怎么连起来的"

1. [01-前后端主链路.md](./01-前后端主链路.md) — 端到端全链路一图看清
2. `ConversationController.java` — UI 层入口，找 `tryExecute` 方法
3. `ConversationApplicationServiceImpl.java` — `chat()` 方法，约 800 行，是调度大厅

### 想搞清楚"DDD 分层怎么理解，怎么改代码"

1. [02-DDD分层架构.md](./02-DDD分层架构.md) — 七层职责与依赖方向

### 想搞清楚"请求鉴权和用户信息怎么传"

## 3. 推荐阅读顺序

**第一轮（全局视角，约 1 天）**

1. 本目录 00 → 01 → 02 → 03（了解整体轮廓和分层套路）
2. `application.yml`（了解配置体系和白名单）
3. `ConversationController.java`（了解接口全景）

**第二轮（执行核心，约 1 天）**

4. `ConversationApplicationServiceImpl.java` 的 `chat()` 方法（800 行，重点看准入检查 + AgentContext 构建 + 心跳逻辑）
5. `AgentExecutor.java`（重点看 `execute()` 方法，组件编排逻辑）
6. `ModelInvoker.java`（Function Calling 循环）
7. 04-模型调用与流式推流.md

**第三轮（专项深入，按需）**

8. 知识库：`KnowledgeBaseSearcher.java` + `app-platform-knowledge` 模块
9. 记忆：`app-platform-memory` 模块
10. 插件/工作流：`PluginExecutor.java` / `WorkflowExecutor.java`
11. MCP：`app-platform-mcp` 模块 + `McpExecutor.java`
12. 计费体系：`app-platform-pricing` + `app-platform-subscription`
