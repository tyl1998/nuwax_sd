# 按仓库学习路线

生成时间：2026-06-10

目标：不是孤立地看单仓库，而是按“入口 -> 主链路 -> 沙箱 -> 端侧补充 -> 生态扩展”的顺序建立整体认知。

## 第一阶段：先看部署入口，拿到总地图

### 1. `nuwax-cli`

学习目标：

- 明白一键部署、升级、回滚、备份是怎么组织的
- 知道生产/测试环境到底由谁拉镜像、谁起服务

建议先看：

- `../nuwax-cli/README.md`
- `../nuwax_deploy.sh`
- `../nuwax_deploy/docker/README.md`

阶段产出：

- 能回答“本地/服务器怎么部署主服务”
- 能回答“升级入口在哪、备份入口在哪”

### 2. `nuwax_deploy` / `nuwax_computer_deploy`

学习目标：

- 拿到主服务与 Agent Computer 的容器编排视图

建议先看：

- `../nuwax_deploy/docker/docker-compose.yml`
- `../nuwax_computer_deploy/docker-computer/docker-compose.yml`

阶段产出：

- 能说清主服务和沙箱服务分别启动哪些容器
- 能说清 `frontend/backend/mcp-proxy/rcoder` 之间的关系

## 第二阶段：吃透平台主链

### 3. `nuwax`

学习目标：

- 前端有哪些核心页面和能力入口
- 它如何对接 `nuwax-backend`

建议先看：

- `../nuwax/README.md`
- 前端应用入口与请求封装
- 与登录、Agent、知识库、插件相关的页面目录

阶段问题：

- 前端的核心模块有哪些
- 哪些功能是平台主功能，哪些是附加功能

### 4. `nuwax-backend`

学习目标：

- 建立整个平台的业务中枢认知
- 知道后端如何串知识库、模型、工作流、插件、记忆、沙箱

建议先看：

- `../nuwax-backend/README.md`
- 配置文件入口
- Web/API/router/controller 层
- 与知识库、插件、Agent、模型相关模块

阶段问题：

- 平台最核心的业务边界是什么
- 哪些能力由后端主控，哪些能力下发到 `mcp-proxy` 或 `rcoder`

### 5. `mcp-proxy`

学习目标：

- 理解 MCP 协议的代理、转换、路由、健康检查
- 理解它为什么是平台扩展能力的中间层

建议先看：

- `../mcp-proxy/README.md`
- `../mcp-proxy/docker/docker-compose.yml`
- server / client / proxy 相关模块

阶段问题：

- MCP 服务是如何注册、转发、保活的
- 后端和客户端为什么都可能依赖它

## 第三阶段：吃透沙箱与执行层

### 6. `rcoder`

学习目标：

- 明白它如何接请求、建容器、起 agent、回传 SSE
- 明白它为什么是 Agent Computer 的主控层

建议先看：

- `../rcoder/README.md`
- `../rcoder/docs/grpc-architecture.md`
- `../rcoder/docs/project-birdseye.md` 若后续重新补回，可优先读

重点关注：

- `router`
- `computer` 相关 handler
- `docker_manager`
- `gRPC client`
- `Pingora proxy`

阶段问题：

- `rcoder` 如何把 HTTP 请求转成容器内执行
- 它怎样管理 VNC、音频、输入法、容器生命周期

### 7. `noVNC`

学习目标：

- 补足远程桌面前端这一块

建议先看：

- `../noVNC/README.md`
- 嵌入方式与代理方式

### 8. `nuwaxcode`

学习目标：

- 理解容器内 Agent 引擎本身如何工作
- 理解它与 ACP、工具调用、代码执行的关系

建议先看：

- 仓库根 `README`
- 服务入口
- 与 `stdio` / `serve` / 工具调用相关实现

阶段问题：

- 它是怎样作为“Agent 引擎”被 `rcoder` 或 `nuwaclaw` 使用的
- 它与通用 `OpenCode` 的差异在哪

### 9. `claude-code-acp-ts`

学习目标：

- 理解 ACP 适配器在多 Agent 架构里的位置

建议先看：

- 仓库根 `README`
- ACP server 启动入口
- tool call / permission / session 相关逻辑

阶段问题：

- 为什么 `rcoder` 默认 agent id 是它
- 它与 `nuwaxcode` 是替代关系还是并行关系

### 10. `run_code_rmcp`

学习目标：

- 理解脚本执行能力如何被暴露成 CLI/MCP 服务

建议先看：

- 仓库根 `README`
- `src/`
- `examples/`

阶段问题：

- 它是平台插件能力的一部分，还是 Agent 工具链的一部分
- 它与 `mcp-proxy` 的关系是什么

## 第四阶段：补上端侧与本地运行时

### 11. `nuwaclaw`

学习目标：

- 理解本地桌面客户端如何组织多 Agent、多依赖、多会话

建议先看：

- `../nuwaclaw/README.md`
- `../nuwaclaw/AGENTS.md`

重点关注：

- 引擎管理
- 本地依赖安装
- 文件服务/MCP/技能同步

### 12. `nuwax-file-server`

学习目标：

- 理解本地文件、工程目录、上传目录和工作目录是怎么被统一管理的

建议先看：

- `../nuwax-file-server/README.md`
- `../nuwax-file-server/docs/ENV.md`

### 13. `nuwax-mobile`

学习目标：

- 补足移动端能力，不影响主链理解时机

建议放在前三阶段完成后再看。

## 第五阶段：最后再看生态扩展仓库

这几类仓库先不影响主部署主链，不建议一开始投入太多时间：

- `xagi-frontend-templates`
- `vite-plugin-design-mode`
- `dev-inject`

适合在你已经看懂：

- 主站前端怎么搭
- 后端怎么编排能力
- 沙箱怎么执行 Agent

之后，再回头看“如何做模板化应用生成和开发态增强”。

## 最推荐的实际学习顺序

1. `nuwax-cli`
2. `nuwax_deploy` / `nuwax_computer_deploy`
3. `nuwax`
4. `nuwax-backend`
5. `mcp-proxy`
6. `rcoder`
7. `noVNC`
8. `nuwaxcode`
9. `claude-code-acp-ts`
10. `run_code_rmcp`
11. `nuwaclaw`
12. `nuwax-file-server`
13. `nuwax-mobile`

## 每看完一个仓库，建议固定回答这 4 个问题

1. 这个仓库在整体链路里扮演什么角色？
2. 它的上游调用方和下游依赖分别是谁？
3. 它暴露了哪些入口：HTTP、CLI、stdio、gRPC、Docker、前端页面？
4. 如果它挂了，平台哪一段会受影响？

按这个口径做笔记，后面你会很快把整个 Nuwax 链路串起来。
