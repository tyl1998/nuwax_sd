# log-platform 阅读路径

## 1. 新人 30 分钟路线

**目标**：理解 log-platform 是什么、怎么跑起来、API 有哪些。

| 步骤 | 看什么 | 预计时间 |
|------|--------|----------|
| 1 | [`00-总览.md`](./00-总览.md) — 全貌：架构图、技术栈、三大 API 域 | 5 min |
| 2 | `README.md` — 官方 README，Quick Start 和 API 速查 | 10 min |
| 3 | `src/main.rs` — 程序入口，启动流程一目了然 | 10 min |
| 4 | `config.yml` — 配置文件长什么样 | 5 min |

30 分钟结束后，你应该能回答：
- log-platform 用什么语言写的？
- 它跟 Quickwit 是什么关系？
- 有哪三大 API 域？
- 本地怎么启动？

## 3. Agent 日志域深入路线

**目标**：深入理解最复杂的 API 域（Agent 日志），包括索引版本管理和数据迁移。

| 步骤 | 看什么 | 预计时间 |
|------|--------|----------|
| 1 | `src/services/agent_log_quickwit.rs` — 完整通读 | 40 min |
| 2 | `src/api/agent_logs.rs` — 完整通读 | 20 min |
| 3 | `src/models/app_state_model.rs` — 全局状态结构 | 5 min |
| 4 | [03-服务层与Quickwit交互](./03-服务层与Quickwit交互.md) — 对照文档理解设计意图 | 10 min |

这条路线重点关注：
- `create_named_index()` — 如何创建不同版本的索引
- `migrate_data_from_old_index()` — 数据迁移的读取-写入-断点续传机制
- `search_agent_logs()` — 搜索查询的构建和结果解析
- `batch_ingest_agent_logs()` — NDJSON 拼接和批量提交

## 4. 知识库域深入路线

**目标**：理解知识库片段的完整生命周期（CRUD + 异步删除 + 统计）。

| 步骤 | 看什么 | 预计时间 |
|------|--------|----------|
| 1 | `src/services/knowledge_quickwit.rs` — 完整通读（最长的服务文件） | 60 min |
| 2 | `src/api/knowledge_logs.rs` — 对应的 handler 层 | 15 min |
| 3 | [02-API接口与路由](./02-API接口与路由.md) — 知识库片段域部分 | 10 min |

这条路线重点关注：
- `delete_knowledge_segments_async()` — 异步删除任务的创建和状态管理
- `clear_all_knowledge_segments()` — Quickwit clear API 的使用
- `update_knowledge_segment()` — 先删后写策略
- `query_segment_ids()` — 按条件查询片段 ID
- `get_knowledge_stats()` — 聚合统计查询
