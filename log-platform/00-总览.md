# log-platform 总览

## 1. 这个服务解决什么问题

log-platform 是平台的**日志管理中间层**，核心职责是：

- 为 Agent 对话日志、知识库片段、通用运行日志提供统一的**写入、检索、删除**能力
- 屏蔽底层 Quickwit（全文搜索引擎）的细节，向上层业务提供简洁的 HTTP API
- 支持日志索引的版本管理与数据迁移（v1 → v2）

一句话：**它是平台所有日志数据的"收发室"，业务方只管往里扔、往里查，不用关心 Quickwit 怎么建索引、怎么做全文检索。**

## 3. 关键入口文件速查

### 启动入口

| 文件 | 说明 |
|------|------|
| `src/main.rs` | 程序入口：加载配置 → 初始化日志 → 创建索引 → 启动 Axum 服务 |
| `src/config/mod.rs` | 配置加载：YAML 文件 + 环境变量覆盖 |

### 路由定义

| 文件 | 说明 |
|------|------|
| `src/api/routes.rs` | 全量路由表：Agent 日志、知识库、通用日志三大域 |
| `src/api/agent_logs.rs` | Agent 日志 API handler（写入/搜索/迁移） |
| `src/api/knowledge_logs.rs` | 知识库片段 API handler（CRUD / 异步删除） |
| `src/api/record_common_logs.rs` | 通用日志 API handler |

### 服务层（与 Quickwit 交互）

| 文件 | 说明 |
|------|------|
| `src/services/agent_log_quickwit.rs` | Agent 日志服务：索引管理、NDJSON 写入、全文搜索 |
| `src/services/knowledge_quickwit.rs` | 知识库服务：片段写入/搜索/更新/删除/统计 |
| `src/services/record_common_log_quickwit.rs` | 通用日志服务 |

### 中间件与模型

| 文件 | 说明 |
|------|------|
| `src/middlewares/mod.rs` | 中间件栈：request_id、请求日志、trace、server_time |
| `src/models/app_state_model.rs` | 全局状态：reqwest Client + QuickwitConfig |

## 4. 技术栈一句话结论

| 层 | 选型 |
|----|------|
| 语言 | Rust (edition 2024) |
| HTTP 框架 | Axum 0.8 + Tower 0.5 |
| 异步运行时 | Tokio 1 |
| HTTP 客户端 | reqwest 0.12 |
| 本地嵌入式存储 | redb 3.1（迁移状态持久化） |
| API 文档 | utoipa 5.4 + Swagger UI |
| 搜索引擎后端 | Quickwit（外部服务，HTTP API 交互） |
| 配置格式 | YAML（serde_yaml） |
| 容器化 | 多阶段 Docker 构建（rust:1.90-slim → debian:bookworm-slim） |

## 5. 三大 API 域一句话结论

| 域 | 前缀 | 核心能力 |
|----|------|----------|
| Agent 日志 | `/api/agent/log/*` | 单条/批量写入、全文搜索、详情查询、索引版本管理、数据迁移 |
| 知识库片段 | `/api/knowledge/*` | 片段 CRUD、BM25 评分搜索、统计、异步删除任务、按 query 清理 |
| 通用日志 | `/api/logs/*` | 日志写入、搜索 |
