# nuwax-cli 阅读路径

这篇不解释细节，只回答一个问题：你现在该从哪开始读。

## 1. 新人 30 分钟路线

适合：

- 刚接触 Nuwax
- 先建立概念，不急着改代码

建议顺序：

1. [00-总览.md](./00-总览.md)
2. [01-新人快速上手.md](./01-新人快速上手.md)
3. [README.zh-CN.md](../../nuwax-cli/README.zh-CN.md)
4. [nuwax-cli/src/cli.rs](../../nuwax-cli/nuwax-cli/src/cli.rs)
5. [nuwax-cli/src/init.rs](../../nuwax-cli/nuwax-cli/src/init.rs)
6. [nuwax-cli/src/commands/status.rs](../../nuwax-cli/nuwax-cli/src/commands/status.rs)

读完应能回答：

1. `nuwax-cli` 是干什么的？
2. 初始化后本地会留下什么？
3. 正常的操作顺序是什么？

1. [02-源码与架构.md](./02-源码与架构.md)
2. [nuwax-cli/src/main.rs](../../nuwax-cli/nuwax-cli/src/main.rs)
3. [nuwax-cli/src/app.rs](../../nuwax-cli/nuwax-cli/src/app.rs)
4. [nuwax-cli/src/commands/mod.rs](../../nuwax-cli/nuwax-cli/src/commands/mod.rs)

## 3. 排障路线

适合：

- 命令跑不通
- 初始化、状态检查、升级流程有问题

建议顺序：

1. [nuwax-cli/src/main.rs](../../nuwax-cli/nuwax-cli/src/main.rs)
2. [nuwax-cli/src/commands/status.rs](../../nuwax-cli/nuwax-cli/src/commands/status.rs)
3. [client-core/src/config.rs](../../nuwax-cli/client-core/src/config.rs)
4. [client-core/src/database_manager.rs](../../nuwax-cli/client-core/src/database_manager.rs)
5. [nuwax-cli/src/commands/docker_service.rs](../../nuwax-cli/nuwax-cli/src/commands/docker_service.rs)
6. [client-core/src/api.rs](../../nuwax-cli/client-core/src/api.rs)

最常见问题分层：

- CLI 参数问题：看 `cli.rs`
- 初始化失败：看 `init.rs`
- 配置/数据库找不到：看 `main.rs` + `config.rs`
- Docker 起不来：看 `docker_service.rs` + `container/`
- 版本检测或下载失败：看 `api.rs` + `check_update.rs`

适合：

- 你最关心这个仓库的“核心价值”
- 要改升级策略、离线部署、增量升级

建议顺序：

1. [docs/UPGRADE_ARCHITECTURE.md](../../nuwax-cli/docs/UPGRADE_ARCHITECTURE.md)
2. [03-关键流程梳理.md](./03-关键流程梳理.md)

2. 为什么 `auto-upgrade-deploy` 比 `upgrade` 更重？
3. 离线下载和离线部署分别在哪一层处理？

## 5. 二次开发路线

适合：

- 要加新命令
- 要改升级、备份、Docker 行为

建议顺序：

1. [cli.rs](../../nuwax-cli/nuwax-cli/src/cli.rs)
2. [app.rs](../../nuwax-cli/nuwax-cli/src/app.rs)
3. [commands/mod.rs](../../nuwax-cli/nuwax-cli/src/commands/mod.rs)
4. 相关命令 handler
5. 对应 `client-core` 模块

加命令的一般路径：

1. 在 `cli.rs` 增加命令定义
2. 在 `commands/` 下增加 handler
3. 在 `app.rs` 的 `run_command` 里接上
4. 在 `client-core` 中补核心逻辑

## 6. 最短结论

如果你只记一条阅读顺序，我建议是：

1. `00-总览`
2. `01-新人快速上手`
3. `cli.rs`
4. `main.rs`
5. `app.rs`
6. `03-关键流程梳理`
7. `upgrade.rs` / `backup.rs` / `docker_service.rs`

这样既不会一上来陷进细节，也不会只停在概念层。
