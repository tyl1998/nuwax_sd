# nuwax-cli 新人快速上手

这篇面向第一次接触 `nuwax-cli` 的人，目标不是让你读懂所有源码，而是先知道：

- 这个工具怎么用
- 正常使用顺序是什么
- 每一步大概在做什么

## 1. 一句话理解

`nuwax-cli` 是 Nuwax 平台本地部署和升级的命令行控制器。

你可以把它理解成：

- “先初始化工作目录”
- “再下载 Docker 服务包”
- “再启动/停止/检查服务”
- “升级前后顺手做备份和回滚”

## 3. 典型使用顺序

最常见路径是：

```bash
nuwax-cli init
nuwax-cli status
nuwax-cli upgrade
nuwax-cli docker-service start
```

更完整一点的路径是：

```bash
nuwax-cli init
nuwax-cli status
nuwax-cli upgrade
nuwax-cli docker-service start
nuwax-cli backup
nuwax-cli auto-upgrade-deploy run
```

## 4. 每个命令大概做什么

### `nuwax-cli init`

作用：

- 创建 `config.toml`
- 创建 `docker/`、备份目录、缓存目录
- 初始化 DuckDB
- 给本地客户端生成并注册 `client_id`

代码入口：

- [main.rs](../../nuwax-cli/nuwax-cli/src/main.rs)
- [init.rs](../../nuwax-cli/nuwax-cli/src/init.rs)

### `nuwax-cli status`

作用：

- 显示 CLI 版本
- 显示当前 Docker 服务版本
- 检查配置文件、compose 文件、服务包文件是否存在
- 尝试做一次 Docker 服务状态检查

代码入口：

- [main.rs](../../nuwax-cli/nuwax-cli/src/main.rs)
- [commands/status.rs](../../nuwax-cli/nuwax-cli/src/commands/status.rs)

### `nuwax-cli upgrade`

作用：

- 查询远端版本
- 判断是全量升级、增量升级还是无需升级
- 下载服务包到本地缓存目录

代码入口：

- [commands/update.rs](../../nuwax-cli/nuwax-cli/src/commands/update.rs)
- [client-core/src/upgrade.rs](../../nuwax-cli/client-core/src/upgrade.rs)

### `nuwax-cli docker-service start`

作用：

- 使用 Docker / Docker Compose 启动本地服务

代码入口：

- [commands/docker_service.rs](../../nuwax-cli/nuwax-cli/src/commands/docker_service.rs)
- [nuwax-cli/src/docker_service/](../../nuwax-cli/nuwax-cli/src/docker_service/mod.rs)

### `nuwax-cli backup`

作用：

- 检查服务状态
- 要求持续运行型服务已停止
- 备份 `docker/data` 和 `docker/app`

代码入口：

- [commands/backup.rs](../../nuwax-cli/nuwax-cli/src/commands/backup.rs)
- [client-core/src/backup.rs](../../nuwax-cli/client-core/src/backup.rs)

### `nuwax-cli auto-upgrade-deploy run`

作用：

- 自动检查版本
- 下载服务包
- 停服务
- 解压与替换
- 启服务并校验

代码入口：

- [commands/auto_upgrade_deploy.rs](../../nuwax-cli/nuwax-cli/src/commands/auto_upgrade_deploy.rs)

## 5. 初始化后会落下哪些本地东西

看 [init.rs](../../nuwax-cli/nuwax-cli/src/init.rs) 可以知道，它至少会创建：

- `config.toml`
- `docker/`
- 备份目录
- 缓存目录
- 下载缓存目录
- DuckDB 数据库文件

这里面最值得先记住的是两个：

- `config.toml`
  作用：记录当前 Docker 服务版本、Docker compose 路径、备份目录、缓存目录等
- DuckDB 数据库
  作用：记录客户端 ID、备份记录、运行过程中的状态信息

## 6. 作为新人，最容易搞混的几点

### `nuwax-cli` 不是业务服务

它只负责管理业务服务，不负责真正提供前端、后端、MCP、沙箱能力。

### `status` 和 `init` 是特殊命令

从 [main.rs](../../nuwax-cli/nuwax-cli/src/main.rs) 可以看到：

- `init` 不依赖完整应用初始化
- `status` 即便应用初始化失败，也会尽量输出基础信息

这说明作者把这两个命令当成“救命入口”和“新手入口”。

### `upgrade` 不等于立刻起服务

它更偏“下载和准备服务包”，真正启动服务通常要交给：

- `docker-service start`
- 或 `auto-upgrade-deploy run`
