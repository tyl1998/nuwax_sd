# nuwax-cli 文档设计说明

生成时间：2026-06-11

## 1. 目标

本目录用于沉淀 `nuwax-cli` 的二次梳理文档，同时满足两类读者：

- 新人：快速理解 `nuwax-cli` 是什么、怎么用、在整个 Nuwax 链路中处于什么位置。
- 研发：快速定位源码入口、核心模块、关键流程和扩展点，便于继续读代码或排障。

文档目标不是替代仓库原有 `README` 和 `docs/`，而是做一层“面向阅读和理解”的导航与重组。

## 2. 设计原则

### 2.1 双视角，但不重复造文档

采用“总览 + 新人线 + 研发线 + 流程线 + 阅读路线”的结构：

- 新人先看用途和常用命令，不被源码细节打断。
- 研发可以直接跳到架构、文件入口、流程调用链。
- 重复基础概念只在一处解释，其他文档通过跳转复用。

### 2.2 一份术语体系

后续文档统一使用以下口径：

- 命令入口层：`nuwax-cli/` 子项目中的 CLI 定义、命令分发、应用入口。
- 业务核心层：`client-core/` 中的升级、备份、数据库、容器、SQL Diff 等能力。
- 部署交互层：与 Docker、镜像、备份目录、远端版本、部署目录打交道的部分。

### 2.3 优先建立“可跳转地图”

文档会优先回答：

1. 这个仓库在整体链路里做什么？
2. 常用命令走到哪里？
3. 关键流程依赖哪些模块？
4. 想继续深入，应该点哪一个文件？

## 3. 文档结构

本目录计划维护以下文件：

### `00-总览.md`

作用：

- 一页讲清 `nuwax-cli` 的定位、核心能力、与部署链路的关系。
- 作为整个目录的索引页。

重点内容：

- 仓库定位
- 核心能力清单
- 目录级结构图
- 文档导航

### `01-新人快速上手.md`

作用：

- 面向第一次接触项目的人。

重点内容：

- 一句话理解项目
- 常用命令及典型顺序
- 初始化、状态检查、升级、启动服务的基本路径
- 遇到问题时先看哪里

### `02-源码与架构.md`

作用：

- 面向研发阅读源码。

重点内容：

- Cargo workspace 结构
- `nuwax-cli` 与 `client-core` 分工
- 命令入口文件
- 核心模块与职责
- 推荐优先阅读的源码文件

### `03-关键流程梳理.md`

作用：

- 把“命令怎么用”和“代码怎么跑”连起来。

重点内容：

- `init`
- `status`
- `upgrade`
- `docker-service`
- `backup` / `rollback`

每个流程统一按以下结构写：

- 用户执行什么命令
- 入口落在哪
- 调用了哪些核心模块
- 会操作哪些外部资源

### `04-阅读路径.md`

作用：

- 给不同角色提供最短阅读路径。

重点内容：

- 新人 30 分钟路线
- 研发半天路线
- 排障路线
- 二次开发路线

## 4. 跳转规则

文档之间遵循以下跳转方式：

- `00-总览.md` 跳所有子文档
- `01-新人快速上手.md` 中提到命令时，跳 `03-关键流程梳理.md`
- `02-源码与架构.md` 中提到模块时，尽量跳具体源码文件
- `03-关键流程梳理.md` 中每条流程同时回链到“怎么用”和“代码在哪”
- `04-阅读路径.md` 作为阅读入口回链前面四篇

## 5. 与仓库原文档的关系

`nuwax-cli` 原仓库已经有这些基础材料：

- `README.md`
- `README.zh-CN.md`
- `CLAUDE.md`
- `docs/UPGRADE_ARCHITECTURE.md`
- `docs/RELEASE_NOTES_v2.0.md`
- `docs/FINAL_REFACTORING_SUMMARY.md`

本目录不会机械搬运原文档，而是：

- 提炼阅读视角
- 重组为更适合理解的顺序
- 必要时引用原文档和源码文件

## 6. 本轮产出范围

本轮先完成首版 5 篇文档的基础内容，重点覆盖：

- 项目定位
- 常用命令
- 模块分工
- 关键流程
- 阅读顺序

暂不展开过深内容：

- 所有命令的逐参数手册
- 所有模块的逐函数注释级拆解
- 每个历史重构版本的完整演进记录

这些后续可以按需继续补。
