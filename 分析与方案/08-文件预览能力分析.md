# skill-details 路由文件预览能力分析

## 1. 背景

分析 `space/:spaceId/skill-details/:skillId` 路由下对以下文件类型的预览支持情况：

`docx` / `doc` / `xlsx` / `xls` / `pptx` / `ppt` / `pdf` / `html` / `md`

---

## 2. 路由与组件关系

```
路由: /space/:spaceId/skill-details/:skillId
页面: src/pages/SkillDetails/index.tsx
核心组件: FileTreeView (isProjectSkill=true)
预览逻辑: FileTreeView → renderContent()
```

数据流：

```
GET /api/skill/{skillId}
  → SkillDetailInfo.files: SkillFileInfo[]
    → transformFlatListToTree()
      → FileNode[] (含 content 和 fileProxyUrl)
        → renderContent() 根据文件类型分支渲染
```

---

## 3. renderContent() 渲染分支顺序

```tsx
// src/components/FileTreeView/index.tsx:1381
const renderContent = () => {
  1. VNC 桌面模式        → viewMode === 'desktop'
  2. 空文件列表          → !files?.length
  3. 无选中文件          → !selectedFileNode
  4. 视频                → isVideo && fileProxyUrl        → FilePreview (video)
  5. 音频                → isAudio && fileProxyUrl        → FilePreview (audio)
  6. Office 文档         → isOfficeDocument && fileProxyUrl → FilePreview (docx/xlsx/pptx/pdf)
  7. JSON               → .json && fileProxyUrl          → FilePreview (text)
  8. 图片                → isImage                        → FilePreview / ImageViewer
  9. 不支持预览          → !isPreviewable || isLink || .doc → "不支持预览"
  10. HTML / Markdown    → viewFileType==='preview' && (fileProxyUrl || content) → FilePreview
  11. 代码 / 文本        → 默认分支                        → CodeViewer (Monaco Editor)
}
```

## 4. 各文件类型支持情况

### 4.1 能否基于 `contents`（文本内容）预览？

| 文件类型 | 能否用 content 预览 | 原因 |
|---------|-------------------|------|
| **md** | ✅ 可以 | 文本格式，`PureMarkdownRenderer` 直接渲染文本 |
| **html** | ✅ 可以 | 文本格式，sandbox iframe 可接受 HTML 文本 |
| **docx** | ❌ 不可以 | 二进制格式，`@js-preview/docx` 需要 URL 或 ArrayBuffer |
| **doc** | ❌ 不可以 | 二进制格式 + 旧格式不被 `@js-preview/docx` 支持 |
| **xlsx** | ❌ 不可以 | 二进制格式，`@js-preview/excel` 需要 URL 或 ArrayBuffer |
| **xls** | ❌ 不可以 | 二进制格式，同 xlsx |
| **pptx** | ❌ 不可以 | 二进制格式，`pptx-preview` 需要 URL → fetch → ArrayBuffer |
| **ppt** | ❌ 不可以 | 二进制格式，同 pptx |
| **pdf** | ❌ 不可以 | 二进制格式，`@js-preview/pdf` 需要 URL 或 ArrayBuffer |

### 4.2 Office 文档预览的技术限制

`FilePreview` 组件内部对 Office 文档的处理（`src/components/business-component/FilePreview/index.tsx:544-601`）：

```typescript
// docx: jsPreviewDocx.init(container).preview(previewSrc)  → 需要 URL 或 ArrayBuffer
// xlsx: jsPreviewExcel.init(container).preview(previewSrc)  → 需要 URL 或 ArrayBuffer
// pdf:  jsPreviewPdf.init(container).preview(previewSrc)    → 需要 URL 或 ArrayBuffer
// pptx: fetch(URL).then(arrayBuffer).then(preview)          → 需要 URL
```

`content` 属性仅在以下类型中使用：
- `markdown` / `text`：直接用 `propsContent` 渲染
- `html`：用 `propsContent` 作为 iframe 内容
- **`docx` / `xlsx` / `pptx` / `pdf`：不使用 `content`，只用 `src`（URL）**

### 4.3 完整支持矩阵

| 文件类型 | 需要 `fileProxyUrl` | 需要 `content` | 当前是否可预览 | 渲染组件 | 第三方库 |
|---------|-------------------|---------------|-------------|---------|---------|
| **docx** | ✅ 必须 | ❌ 不支持 | ⚠️ 取决于后端是否返回 fileProxyUrl | FilePreview | `@js-preview/docx` |
| **doc** | - | - | ❌ 不支持 | 显示"不支持预览" | `@js-preview/docx` 不支持旧格式 |
| **xlsx** | ✅ 必须 | ❌ 不支持 | ⚠️ 取决于后端是否返回 fileProxyUrl | FilePreview | `@js-preview/excel` |
| **xls** | ✅ 必须 | ❌ 不支持 | ⚠️ 取决于后端是否返回 fileProxyUrl | FilePreview | `@js-preview/excel` |
| **pptx** | ✅ 必须 | ❌ 不支持 | ⚠️ 取决于后端是否返回 fileProxyUrl | FilePreview | `pptx-preview` |
| **ppt** | ✅ 必须 | ❌ 不支持 | ⚠️ 取决于后端是否返回 fileProxyUrl | FilePreview | `pptx-preview` |
| **pdf** | ✅ 必须 | ❌ 不支持 | ⚠️ 取决于后端是否返回 fileProxyUrl | FilePreview | `@js-preview/pdf` |
| **html** | 可选 | ✅ 可以 | ✅ 支持 | FilePreview (iframe) | 原生 |
| **md** | 可选 | ✅ 可以 | ✅ 支持 | FilePreview (PureMarkdownRenderer) | `ds-markdown` |

## 6. 总结

### 当前技能详情页可预览的文件类型

- ✅ **文本类**：md、html、代码文件（js/ts/py/java/go 等 30+ 种）、txt、json、yaml、csv 等 — 通过 `contents` 文本内容渲染
- ✅ **图片类**：jpg/png/gif/webp/svg/bmp/ico — 通过 `fileProxyUrl` 或 `content`（Base64）渲染

### 需要后端返回 `fileProxyUrl` 才能预览的文件类型

- ⚠️ **Office 文档**：docx、xlsx、xls、pptx、ppt、pdf — 二进制格式，必须通过 URL 获取二进制数据

### 明确不支持的文件类型

- ❌ **doc**（旧版 Word）：`@js-preview/docx` 库不支持，代码中显式排除

### 核心结论

技能文件的数据存储模型是将文件内容以**文本字符串**存储在 `contents` 字段中。这种模型天然支持文本类文件（代码、Markdown、HTML），但**无法支持 Office 文档等二进制文件**。Office 文档的预览依赖后端返回 `fileProxyUrl`（指向二进制文件的 HTTP 下载地址），仅靠 `contents` 文本内容无法实现预览。
