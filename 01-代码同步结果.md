## 3. 缺失仓库补齐策略

对于本地缺失、但在部署链路或运行时依赖中明确出现的仓库，采用两种方式补齐：

1. 优先尝试标准 Git 拉取。
2. 若仓库默认分支特殊或普通 clone 不稳定，则直接从 GitHub 最新分支源码归档拉取快照。

这样可以保证“代码先到本地”，不被远端默认分支命名或长历史阻塞。
