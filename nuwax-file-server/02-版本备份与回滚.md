# nuwax-file-server：版本备份与回滚

项目版本管理通过 zip 快照实现：备份时把项目目录（排除 node_modules 等大目录）压缩成 zip 存到 `UPLOAD_PROJECT_DIR`，回滚时用 zip 覆盖还原，保留被排除的目录不动。

## 1. 核心函数（backupUtils.js）

| 函数 | 说明 |
|------|------|
| `backupProjectToZip(projectId, projectPath, outZipPath)` | 把项目目录压缩为 zip |
| `restoreProjectFromZip(projectId, projectPath, zipPath)` | 从 zip 还原项目目录 |
| `pruneMissingFiles(baseDir, keepRelativePaths, excludeDirNames)` | 删除不在新版本列表中的文件 |
| `removeEmptyDirectories(baseDir, excludeDirNames)` | 后序遍历删除空目录 |

## 2. 备份流程

```
backupProjectToZip()
  │
  ├── 创建临时目录 UPLOAD_PROJECT_DIR/{projectId}/backup_temp_{ts}/
  │
  ├── copyDirectoryFiltered(projectPath → tempDir)
  │     排除 TRAVERSE_EXCLUDE_DIRS（node_modules、.git 等）
  │     排除 BACKUP_TRAVERSE_EXCLUDE_FILES（特定文件）
  │
  ├── archiver("zip", level:9) 压缩 tempDir → outZipPath
  │
  └── finally: rm -rf tempDir（无论成败清理临时目录）
```

**为什么先复制再压缩而不是直接压缩源目录？**  
中间经过 `copyDirectoryFiltered` 过滤，可以精确排除大目录，避免 archiver 在迭代时跳过目录产生不一致，也让压缩和过滤逻辑解耦。

## 3. 回滚流程

```
restoreProjectFromZip()
  │
  ├── 读取 projectPath 根目录条目
  │     跳过 excludeDirNames（node_modules 等）
  │     跳过 excludeFileNames（特定文件）
  │     ← 其余全部 rm -rf（清空现有内容）
  │
  └── yauzl 流式解压 zipPath → projectPath
        ├── 目录条目：mkdir -p
        ├── 文件条目：openReadStream → pipe → writeStream
        └── 路径穿越防护：resolvedTargetPath.startsWith(resolvedProjectPath + sep)
```

**路径穿越防护**：解压前校验 `path.resolve(targetPath)` 必须以项目目录作前缀，防止恶意 zip 中的 `../../etc/passwd` 类路径覆盖系统文件。

**为什么用 yauzl 而不是系统 unzip？**  
容器环境不保证有 `unzip` 命令，且 yauzl 是流式处理，内存占用低；archiver 同理。

## 4. AI Agent 代码更新（all-files-update）

Agent 修改代码时不走 zip，而是直接调用 `all-files-update` 接口，底层用 `pruneMissingFiles` + `removeEmptyDirectories` 实现"增量覆盖"语义：

```
all-files-update 请求体：{ files: [{ path, content }, ...] }
  │
  ├── 把新版本所有文件路径收集为 keepRelativePaths: Set
  │
  ├── pruneMissingFiles(projectPath, keepRelativePaths, excludeDirs)
  │     遍历现有文件，不在 keepRelativePaths 中 → 删除
  │     但保护：
  │       ├── 隐藏文件（. 开头）
  │       └── CONTENT_TRAVERSE_EXCLUDE_FILES（AGENT.md、CLAUDE.md 等）
  │
  ├── 写入新版本所有文件内容
  │
  └── removeEmptyDirectories(projectPath, excludeDirs)
        后序遍历，删除空目录
```

**为什么保护 AGENT.md / CLAUDE.md？**  
这些是 AI Agent 的系统提示配置文件，由平台注入，不应被 Agent 自身的文件覆盖操作删除。

## 5. 版本管理 API 速查

| 接口 | 说明 | 底层 |
|------|------|------|
| `POST /api/project/backup-current-version` | 备份当前版本为 zip | `backupProjectToZip` |
| `POST /api/project/rollback-version` | 回滚到历史版本 | `restoreProjectFromZip` |
| `GET /api/project/get-project-content-by-version` | 读取历史版本内容 | 解压 zip 读文件 |
| `POST /api/project/all-files-update` | Agent 全量更新代码 | `pruneMissingFiles` + 写文件 |
| `POST /api/project/specified-files-update` | 更新指定文件 | 直接写文件 |

## 6. 排除目录配置

通过 `appConfig/index.js` 从环境变量读取，典型值：

```
TRAVERSE_EXCLUDE_DIRS         = ["node_modules", ".git", "dist", ".next"]
BACKUP_TRAVERSE_EXCLUDE_FILES = [".env", ".env.local"]
CONTENT_TRAVERSE_EXCLUDE_FILES= ["AGENT.md", "CLAUDE.md"]
```

备份、回滚、内容读取三个场景各有独立的排除列表配置，精度粒度到文件名。

## 一句话总结

版本备份走"过滤复制 → archiver 压缩 → 清理临时目录"三步，回滚走"清空现有文件（保留排除项）→ yauzl 流式解压"两步，路径穿越防护和 AI 配置文件保护是两个关键安全边界；Agent 更新代码走 `pruneMissingFiles` 增量语义而非 zip，效率更高。
