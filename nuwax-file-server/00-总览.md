# nuwax-file-server 总览

`nuwax-file-server` 是一个 **本地/容器文件服务**，用 Node.js（ES Modules）编写，基于 Express。它管理 AI Agent 的项目工作目录、构建流程、文件上传下载，并为 Computer Agent 容器内的工作空间提供统一的文件操作接口。

一句话定位：`nuwax-file-server` = **项目文件管理 + 前端构建管理服务**，运行在容器或本地机器上，是 nuwaclaw 桌面客户端和 Computer Agent 工作区的文件基础设施。

## 1. 架构概览

```
nuwax-file-server（Express :60000）
├── /api/build     构建管理（dev server 启停、日志）
├── /api/project   项目文件管理（上传、下载、内容读写）
├── /api/project   代码版本管理（code routes 共用此前缀）
├── /api/computer  Computer 工作空间管理（文件列表、上传、下载）
└── /health        健康检查
```

## 3. 关键概念

### 项目路径与隔离

通过 `extractIsolationContext` + `resolveProjectPath` 工具函数解析隔离上下文（`isolation_type` / `tenant_id` / `space_id` / `project_id`），把请求路由到正确的工作目录：

```
PROJECT_SOURCE_DIR/
└── {tenant_id}/
    └── {space_id}/
        └── {project_id}/   ← 实际工作目录
```

与 rcoder 的隔离类型（`tenant`/`space`/`project`）对应，实现多租户文件隔离。

### 端口池（portPool）

管理 dev server 使用的端口，避免多项目并发构建时端口冲突。

### 日志缓存（logCacheManager）

把 dev server 和构建日志缓存到内存，供 `get-dev-log` 接口实时拉取，不依赖文件 watch。

## 4. 配置（环境变量）

| 变量 | 默认值 | 说明 |
|------|--------|------|
| `PORT` | `60000` | 监听端口 |
| `PROJECT_SOURCE_DIR` | — | 项目工作目录根路径 |
| `UPLOAD_PROJECT_DIR` | — | 上传项目压缩包临时目录 |
| `DIST_TARGET_DIR` | — | 构建产物输出目录 |
| `COMPUTER_WORKSPACE_DIR` | — | Computer Agent 工作空间根目录 |
| `INIT_PROJECT_DIR` | — | 项目模板目录 |
| `REQUEST_BODY_LIMIT` | `2000mb` | 上传文件大小限制 |
| `NODE_ENV` | `development` | 环境模式 |

## 5. CLI 管理

```bash
nuwax-file-server start    # 启动（守护进程，PID 文件管理）
nuwax-file-server stop     # 停止
nuwax-file-server restart  # 重启
nuwax-file-server status   # 查看状态
nuwax-file-server start --port 8080 --env production
```

## 6. 与平台的关系

| 调用方 | 调用什么 | 目的 |
|-------|---------|------|
| `nuwaclaw` 桌面客户端 | 构建/项目管理 API | 管理本地项目文件和 dev server |
| `nuwax-backend` | Computer 工作空间 API | 读写 Computer Agent 的文件成果 |
| `rcoder` | 通过容器挂载（无直接 HTTP）| 工作目录挂载到容器内 `/home/user` |
| `nuwax` 前端 | 通过 nuwax-backend 间接使用 | 预览/下载项目文件 |

## 一句话总结

`nuwax-file-server` 是平台的文件管理基础设施，提供项目文件 CRUD、版本管理、前端构建 dev server 管理、Computer Agent 工作空间管理等能力，多租户路径隔离和端口池管理使其能安全支撑并发的多项目场景。
