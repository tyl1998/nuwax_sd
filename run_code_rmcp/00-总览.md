# run_code_rmcp 总览

`run_code_rmcp` 是一个 **代码隔离执行工具**，用 Rust 编写，支持两种调用方式：CLI 工具（`run_code_rmcp`）和 MCP 服务器（`script_runner`）。它把 JavaScript/TypeScript（Deno 运行时）和 Python（uv 运行时）代码执行能力暴露出来，供 AI Agent 和 nuwax-backend 的代码执行功能使用。

一句话定位：`run_code_rmcp` = **JS/TS/Python 代码安全执行层**，可以作为 MCP 工具注册到 Agent，也可以通过 `POST /run_code` 直接被 mcp-proxy 调用。

## 1. 两种使用方式

| 方式 | 二进制 | 协议 | 适用场景 |
|------|--------|------|---------|
| CLI 工具 | `run_code_rmcp` | 命令行 | 直接在终端测试执行 |
| MCP 服务器 | `script_runner` | MCP over stdio | 注册为 MCP 工具供 Agent 调用 |

```bash
# CLI 直接执行
run_code_rmcp js -f script.js -p '{"name":"World"}'

# 启动 MCP 服务器（stdio 模式）
script_runner --verbose
```

MCP 服务器通过 stdin/stdout NDJSON 通信（rmcp crate），可被 mcp-proxy 用 `command` 类型注册为 stdio MCP 服务。

## 2. 支持的语言与运行时

| 语言 | 运行时 | 特性 |
|------|--------|------|
| JavaScript | Deno | ESM + CommonJS，沙箱隔离 |
| TypeScript | Deno | 原生 TS 执行，无需编译步骤 |
| Python | uv | 自动检测并安装依赖，PEP 696+ |

### Python 依赖自动处理

Python 代码中直接 `import requests` 即可，`run_code_rmcp` 用 `python_imports.pest`（PEG 解析器）分析导入语句，调用 `uv` 自动安装缺少的包：

```python
import requests   # 自动 uv pip install requests
import pandas as pd

def handler(args):
    return pd.DataFrame(requests.get("...").json()).to_dict()
```

## 3. 代码约定

执行的代码需要符合一定约定（以 JS 为例）：

```javascript
// 定义 handler 函数作为入口（可选，无则执行全文）
function handler(input) {
    // input 是 -p 传入的 JSON 参数
    console.log("日志走 logs 字段");
    return "结果走 result 字段";
}
```

返回格式统一为：
```json
{
  "logs": ["console.log 输出"],
  "result": "handler 返回值",
  "error": null
}
```

日志（`console.log`）和返回值分离，方便 Agent 提取结构化结果。

## 4. 缓存机制

用 **blake3** 哈希对代码内容和参数做摘要，相同代码+参数直接返回缓存结果，避免重复执行。`--clear-cache` 参数清除缓存。

## 5. MCP 服务器模式（script_runner）

```
script_runner（MCP stdio 服务器）
├── CodeRunnerService（rmcp ServiceExt 实现）
├── 暴露 MCP 工具：run_js / run_ts / run_python
└── 通信：stdin 接收 MCP 请求 → 执行代码 → stdout 推送 MCP 响应
```

mcp-proxy 中注册方式：

```json
{
  "mcpServers": {
    "code-runner": {
      "command": "script_runner",
      "args": []
    }
  }
}
```

注册后 Agent 可直接调用 `run_js`、`run_ts`、`run_python` 工具。

## 6. 与平台的关系

| 调用方 | 调用方式 | 目的 |
|-------|---------|------|
| nuwax-backend 代码执行功能 | `POST /run_code`（mcp-proxy 直接暴露此接口）| 执行用户提交的代码片段 |
| AI Agent（claude-code-acp-ts 等）| MCP 工具调用（通过 mcp-proxy 注册的 script_runner）| Agent 执行动态生成的代码 |
| 本地开发/测试 | CLI `run_code_rmcp` | 直接验证脚本 |

mcp-proxy 的 `run_code_handler` 实际是把 `POST /run_code` 请求转发给内嵌的 `run_code_rmcp` 逻辑，不需要单独部署。

## 7. 环境预热（warm_up_all_envs）

`warm_up_all_envs()` 预先安装常用 Python 和 JS/TS 依赖，避免首次执行时等待：

```bash
# 在 mcp-proxy main.rs 启动时调用
warm_up_all_envs(None, None, None, None).await?;
```

## 一句话总结

`run_code_rmcp` 是平台的代码安全执行层，通过 Deno（JS/TS）和 uv（Python）运行时提供隔离执行能力，既作为 CLI 工具供开发调试，又作为 MCP stdio 服务（`script_runner`）供 AI Agent 通过 mcp-proxy 调用，blake3 缓存和依赖自动安装使重复执行高效。
